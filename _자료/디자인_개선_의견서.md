## 1. 디자인 컨셉 확정 (P1)

### 컨셉: **"GOLD QUEST — 수집형 게임 아카데미"**
밝은 아이보리·인디고 바탕 위에 **골드 메탈릭**을 유일한 프리미엄 포인트로 쓰는 모바일 수집 게임 문법. 배지 = 금메달 수집, 라이선스 = 골드 인증 카드, 도장 = 빨간 인장 클라이맥스.

**근거 (3줄):**
1. 이미 보유한 에셋이 전부 **금테 메달 + 빨간 인장 + 방패 엠블럼**이다. UI가 이 골드/엠블럼 아이덴티티를 따라가야 에셋과 스킨이 한 몸으로 보인다. 지금은 보라-핑크 오로라 스킨 위에 골드 에셋이 얹혀 있어 서로 겉돈다.
2. 초등 고학년이 실제로 쓰는 상용 앱(브롤스타즈·쿠키런·토스 만보기류)의 화려함은 "배경이 화려한 것"이 아니라 **보상 순간(획득·발급)에 화력을 몰아주는 것**이다. 발주자의 "발산형·실제 앱 같음" 요구를 이 방식으로 충족한다.
3. 폼 입력·퀴즈 독해가 절반인 앱이므로 바탕은 밝게 유지하고, 다크+골드 프리미엄은 **라이선스 카드 한 곳**에만 허용한다(현재 구조와 호환 → 전면 재작업 없이 도달 가능).

**한 줄 요약: 배경은 정돈, 골드는 통일, 화려함은 보상 순간에 집중.**

## 3. 화면별 개선 지시

### 3-0. 공통 (P1)

1. **스타일 레이어 통합**: `<style>` 3개 + 인라인 스타일이 `!important`로 서로 덮어쓰는 구조다. 최종 토큰 기준으로 1개 `<style>`로 병합하고 `!important` 전량 제거. (모든 화면 지시의 전제 작업)
2. **배경 정돈**: 블롭 3개 → 2개(인디고 1 + 골드 1)로 줄이고 `opacity .55 → .35`. 떠다니는 별 파티클 10개 → 6개, 시작·발급 화면에서만 표시(`#screen1~3` 활성 시 `display:none`). 읽기 화면에서 배경이 시선을 끌면 안 된다.
   ```css
   body { background: linear-gradient(180deg,var(--bg-top),var(--bg-bottom)) fixed; }
   ```
3. **진행 바 → 게임 HUD화**: 좌측에 `license_logo.png`(방패 엠블럼, 28px) + 앱명 "AI 탐험가"를 로고로 상시 표시(현재 이 에셋은 로드 선언만 되고 어디에도 안 쓰임 — 반드시 투입). 스텝 원은 현재 이모지 그대로 두되, `done` 상태에 골드 체크 `background:var(--grad-gold)` 적용. 모바일에서 라벨 텍스트는 활성 스텝만 노출.
4. **버튼 위계**: 화면당 `btn-primary`(그라디언트+shine)는 1개만. 나머지는 `btn-secondary`(흰 배경+인디고 테두리). 현재 발급 화면의 "다시 만들기/갤러리 올리기"처럼 primary가 2개 서는 상황 금지.

### 3-1. 시작 화면 `#screen0` (P1)

목표: "학습 자료 표지"가 아니라 **게임 타이틀 스크린**.

    transform:translateX(-50%); width:340px; opacity:.10; z-index:0;
    filter:saturate(0) brightness(1.2); pointer-events:none; }

    -webkit-background-clip:text; color:transparent;
    filter:drop-shadow(0 2px 0 rgba(201,126,16,.35)); }

    box-shadow:0 14px 34px -6px rgba(91,77,240,.6); } }

### 3-3. 설계 폼 `#screen2` (P1 구조 / P2 장식)

목표: **"내 카드를 조립하는 공방"** — 입력할수록 미리보기가 진화하는 느낌을 극대화.

- **P1 — 미리보기 카드를 라이선스 카드와 동일 스킨으로**: 현재 미리보기(보라 그라디언트)와 최종 라이선스(네이비 홀로그램)가 달라서 "실시간 완성" 감이 끊긴다. `--navy-card` + 골드 테두리 + 방패 엠블럼 워터마크를 넣은 **미니 라이선스**로 통일. 획득 배지도 이모지가 아닌 메달 PNG 20px로.
- **P1 — 입력 완성도 게이지**: 미리보기 카드 상단에 `카드 완성도 3/7` 프로그레스 필(pill) 추가. 필드 하나 채울 때마다 골드로 차오름 (`updatePreview()`에서 채워진 필드 수만 세면 됨). 폼을 게임 진척으로 번역하는 최소 장치.
- **P1 — 인풋 상태**: focus 시 `border-color:var(--primary)` + `box-shadow:0 0 0 4px rgba(91,77,240,.14)`. 값이 채워진 인풋은 우측에 `--success` 체크 아이콘 (`background-image`로 처리).
- **P1 — 안전 방패 체크박스 → 방패 칩**: 네이티브 체크박스 숨기고 카드형 토글로. 선택 시 골드 테두리 + 좌측 방패 아이콘 컬러 인, 체크 순간 `transform:scale(1.05)→1` 바운스. 2개 이상 선택 조건은 칩 그룹 상단에 `방패 0/2` 카운터로 상시 표시(토스트로만 알리지 말 것).
  ```css
  .shield-item { border-radius:var(--r-md); min-height:64px; }
  .shield-item:has(input:checked){ border:2.5px solid var(--gold);
    background:linear-gradient(160deg,#FFFDF4,#FFF6DF);
    box-shadow:0 6px 16px -6px rgba(245,166,35,.4); }
  ```
- **P2 — 콤보박스**: 열림 리스트 항목 높이 44px 이상 확보(현재 11px 패딩 ≈ 40px, 소폭 증가). 선택된 항목에 체크 표시.
- **P2 — 모바일**: 640px 이하에서 미리보기 카드를 폼 아래가 아니라 **하단 고정 바(접힌 상태 56px, 탭하면 펼침)**로. 입력 → 즉시 반영 확인 루프가 모바일에서도 살아야 한다.

### 3-4. 다짐·서명 `#screen3` (P2)

목표: **의식(儀式)의 무게감** — 화려함보다 "서약" 연출.

- **다짐 문장 박스**: 보라 점선 → **증서 스타일**로. 아이보리 배경 `#FFFdf5` + 골드 이중 테두리(`border:3px double var(--gold-deep)`) + 좌우 상단 모서리에 방패 엠블럼 24px. 빈칸이 채워지는 순간 채운 어절에 골드 하이라이트 sweep(0.6s 1회).
- **예시 칩**: 선택된 칩은 `--grad-primary` 배경 + 흰 글자로 상태 유지 (현재는 눌러도 선택 흔적이 안 남음 — `aria-pressed` 관리 포함).
- **서명 캔버스**: 점선 회색 박스 → 서명대 연출. 배경에 옅은 가이드 라인 1줄 + 좌하단 `X` 표시(서류 서명선 관용 표현). 펜 굵기 2.5 → 3.5px(어린이 손글씨 가시성). 서명 시작 순간 placeholder가 fade-out.
  ```css

    linear-gradient(#FFFEFA,#FFFEFA) padding-box,
    repeating-linear-gradient(0deg,transparent 0 118px,#E9E2C8 118px 120px); }
  ```
- **발급 버튼**: 이 화면의 클라이맥스이므로 골드 그라디언트 버튼으로 예외 허용(`--grad-gold` + 다크 텍스트). 서명 전에는 `disabled` 시각 상태(채도 낮춤)로 두고, 서명이 감지되면 활성화 + 1회 바운스 — "서명해야 열린다"를 버튼 상태로 말할 것.

목표: 이 앱의 스크린샷 한 장 = 이 화면. **실물 카드 밀도**로 끌어올린다.

- **카드 골격 유지 + 디테일 보강** (네이비+홀로그램 방향은 옳음):
  1. 헤더의 보라 그라디언트 → `--navy-card` 연속 + 하단에 골드 헤어라인 2px. 헤더 좌측에 `license_logo.png` 방패 44px 배치, `🚀 REPUBLIC OF…` 이모지 제거하고 Montserrat 라벨만.
  2. 카드 배경에 **기요셰(guilloché) 패턴** — 지폐/신분증 질감. 이미지 없이 CSS로:
     ```css
     .license-body::before { content:''; position:absolute; inset:0; opacity:.05;
       background: repeating-radial-gradient(circle at 30% 20%, #FFD65C 0 2px, transparent 2px 14px),
                   repeating-radial-gradient(circle at 80% 90%, #A5B4FC 0 2px, transparent 2px 18px);
       pointer-events:none; }
     ```
  3. **획득 배지 이모지 → 메달 PNG 36px** (`lic-badges`, `buildLicenseCard()`에서 img 태그 생성). 미리보기·갤러리와 함께 3곳 모두 PNG로 통일.
  4. **일련번호·발급일 필드 추가**: `NO. AIX-2607-0421`(랜덤 4자리) + 발급일. Montserrat, 골드 컬러. 실물감을 만드는 건 이런 관용 요소다.
  5. 홀로그램 원형 스티커(현재 `::after`)는 유지하되 우상단 → 우하단 인장 옆으로 이동(우상단은 방패와 충돌).
  6. 3D 틸트는 유지, 단 `rotate` 최대치 7°→5°로 (텍스트 가독 흔들림 완화). 터치 기기에서는 틸트 대신 1회 flip-in 진입 연출.
- **발급 시퀀스 연출(P1)**: 현재 "도장+색종이 동시"를 단계 연출로 재배열 — ① 카드가 아래에서 slide-up + 살짝 overshoot(0.5s) → ② 0.4s 후 인장 슬램(`stamp_seal.png` + 화면 미세 shake 4px 1회) → ③ 인장 착지와 동시에 confetti + fanfare. 순차 타이밍만으로 체감 화려함이 배가된다.
- **저장 유도(P2)**: 카드 아래 안내 문구 "📸 스크린샷으로 저장해 선생님께 보여 주세요" 추가 (이미지 다운로드 구현은 P3, html2canvas 도입 시).

### 3-6. 갤러리 (P2)

- **갤러리 카드 = 미니 라이선스**로 통일: 랜덤 원색 그라디언트 6종 → `--navy-card` 베이스 + 액센트 컬러 스트립(상단 6px)만 6종 랜덤. 배지 이모지 → 메달 PNG 18px.
- 각 카드 우하단에 인장 PNG 28px 워터마크 — "발급된 카드"라는 통일된 정체성.
- 새 카드 추가 시 slide-in + 골드 글로우 1.2s 1회 후 소멸 (내 카드가 어디 붙었는지 즉시 식별).
- 빈 갤러리 상태(P3): `maru_cheer.png` + "첫 번째 탐험가가 되어 보세요!" empty state.

## 4. 모션 설계

원칙: **보상 > 피드백 > 전환 > 장식** 순으로 화력 배분. 장식 모션은 총량 축소.

| 우선 | 순간 | 연출 | 지속 |
|---|---|---|---|
| **P1** | 배지 획득 | 팝업: 메달 PNG scale(0→1.15→1) 바운스 + 방사형 골드 버스트(기존 conic 유지) + 획득음. 관문 트랙의 해당 칸으로 메달이 **날아가 안착**(FLIP, 0.5s) | 1.8s |
| **P1** | 라이선스 발급 | §3-5 3단 시퀀스(카드 등장 → 인장 슬램+shake → confetti+fanfare) | 2.2s |
| **P1** | 정답 | 피드백 popIn + 선택지 골드 테두리 플래시 1회 + maru_cheer 등장 | 0.5s |
| **P1** | 오답 | shakeX(현행) + 주황 힌트. 추가 연출 금지 — 오답을 재미있게 만들지 말 것 | 0.45s |
| **P2** | 화면 전환 | 현행 screenIn 유지, 0.45s → 0.35s로 단축 | 0.35s |
| **P2** | XP/완성도 게이지 | 차오를 때 끝단 골드 스파클 1회 (현재 무한 shine 루프는 제거) | 0.6s |
| **P2** | 6관문 완료 | 메달 아치 순차 등장(stagger 80ms) + confetti | 1.5s |
| **P3** | 배경 블롭·별 | §3-0 축소안. 획득 배지의 `earnedGlow`·`ringSpin` 무한 루프는 **첫 3초만 재생 후 정지** | ambient |

- 무한 루프 애니메이션은 CTA pulse, XP shine, 배지 glow, 포일 타이틀 중 **화면당 1개만** 허용.
- `prefers-reduced-motion` 처리는 이미 있음 — 유지.

## 5. 게임화 요소 추가 제안 (3건)

| 제안 | 내용 | 난이도 |
|---|---|---|
| **① 라이선스 등급제** | 무오답 = GOLD, 오답 1~2회 = SILVER, 3회+ = GREEN 라이선스. 카드 테두리·타이틀 포일 색이 등급을 따라감. 기존 `_wrongCount`가 이미 있어 분기만 추가하면 됨. 재도전("다시 만들기") 동기 발생 | **하** |
| **② 관문 연출 강화 — 메달 비행** | 배지 획득 팝업의 메달이 닫힐 때 상단 관문 트랙 슬롯으로 날아가 박히는 FLIP 애니메이션. "모은다"는 감각을 신체화하는 단일 최고 효율 연출 | **중** |
| **③ 탐험가 칭호 생성** | 발급 시 선택한 AI 기능+방패 조합으로 칭호 자동 부여("데이터를 지키는 예측 탐험가")를 카드 이름 위에 표기. 조합 룩업 테이블만으로 구현, 아이들 간 카드 비교 대화 유발 | **하** |

> 셋 모두 수업 시간(1차시) 내 완주를 방해하지 않는 선. 일일 출석·랭킹·타이머류는 수업 도구 성격상 배제했다.

## 6. 하지 말 것 (어린이 UX 가드레일)

1. **다크 배경에 본문·입력 금지** — 다크+골드는 라이선스 카드(및 미니 카드) 한정. 폼·퀴즈·다짐 화면은 반드시 밝은 바탕.
2. **본문 텍스트 16px 미만 금지, 대비 4.5:1 미만 금지.** 글래스 카드 위 텍스트는 배경 블롭이 비치므로 카드 α를 .74 미만으로 내리지 말 것.
3. **터치 타깃 44×44px 미만 금지** — 콤보 리스트 항목, 예시 칩, 서명 지우기 버튼, sfx 토글 포함 전수 확인.
4. **문제 읽는 영역에 움직이는 요소 금지** — 퀴즈 카드 뷰포트 안에서 파티클·shine·glow 무한 루프 불가. 독해 방해 = 학습 실패.
5. **오답에 빨강·부정 연출 금지** — 주황 힌트 + 재도전 톤 고정. 실패음도 현행 수준 이상으로 무겁게 만들지 말 것.
6. **점멸 3회/초 이상 금지, 효과음 기본 볼륨 상향 금지** — 광과민성·교실 환경. 음소거 토글은 항상 접근 가능 위치 유지.
7. **한 화면에 primary 버튼 2개 금지, 폰트 3종 초과 금지, 무한 루프 모션 화면당 1개 초과 금지.**
8. **이모지·PNG 혼용 금지** — 배지가 어떤 화면에서는 이모지, 어떤 화면에서는 메달 PNG로 나오면 수집물의 정체성이 깨진다. 배지 표시 5개 지점(시작/퀴즈/미리보기/라이선스/갤러리) 전부 PNG로 통일.
9. **텍스트를 3D 틸트·회전 위에 장시간 두지 말 것** — 틸트 각도 5° 상한.
10. **입력 검증을 토스트로만 알리지 말 것** — 카운터·인라인 상태로 상시 노출(§3-3 방패 카운터, §3-4 발급 버튼 disabled 상태).
