# 19강 체험 웹 설계서 — 내 맘대로 AI 조종하기 (창의성 조절·Temperature)

- 경로: `/19/index.html` (단일 파일, 바닐라 JS, 서버·API 키 없음)
- 포인트 컬러: **purple** (`--purple: #A78BFA`, `--purple-light: #EDE9FE`)
- 콘셉트: LLM 없이 **'다음 단어 고르기' 확률 시뮬레이터**로 Temperature의 원리를 몸으로 익힌다.
- 핵심 메시지: "AI는 다음에 올 말을 확률로 골라요. 창의성 값은 그 '고르는 폭'이에요. 낮음=안정, 높음=새로움(+엉뚱함). 마지막 확인은 사람이 해요."

### ② 확률 실험실 (개념 체험)
화면 구성(위→아래):
1. **시나리오 탭**: 문장 3개 중 선택(칩 버튼). 기본은 교재 문장 S1.
2. **문장 카드**: `달나라 토끼 토토는 상자를 열자, 안에서 [ ❓ ]가 나왔습니다.` 빈칸은 점선 박스, 뽑히면 단어가 통통 튀며 들어감(스케일 애니메이션).
3. **창의성 슬라이더**: `<input type="range" min="0" max="100">`. 좌측 라벨 "낮음 🧊 안정", 우측 "높음 🔥 새로움". 현재 구간명 표시: 0~33 "낮음", 34~66 "보통", 67~100 "높음". 슬라이더 썸은 44px 이상.
4. **후보 확률 막대(5개)**: 후보 단어 + 가로 막대 + %. `input` 이벤트마다 즉시 재계산·CSS transition 0.25s로 부드럽게 변화. 막대 색은 창의성 등급별(익숙함=sky, 조금 새로움=mint, 새로움=amber, 엉뚱함=purple).
5. **[문장 만들기] 버튼**(큰 실행 버튼, purple) + [연속 5번 만들기] 보조 버튼.
6. **관찰 노트**(`.note`): "슬라이더를 왼쪽 끝으로 → 어떤 단어가 자주 나오나요? 오른쪽 끝으로 → '달빛 고래'도 나오나요? **같은 설정이어도 매번 같지는 않아요.**"

상호작용 흐름:
- [문장 만들기] 클릭 → 4절의 룰렛 연출 → 뽑힌 단어가 빈칸에 들어가고, 최근 뽑기 이력 칩(최대 8개)이 문장 아래에 쌓임. 낮음 설정에서는 이력이 비슷비슷("편지, 편지, 지도…"), 높음에서는 알록달록 — '매번 달라짐'과 '폭의 차이'가 이력으로 보인다.
- [연속 5번 만들기] → 0.4s 간격으로 5회 자동 뽑기(이력 칩이 촤르륵 쌓임). "차이는 여러 답을 비교할 때 잘 보여요"(교재 AI 실험 팁)를 그대로 체험.
- 슬라이더를 움직이면 이력은 유지하되 구분선 칩("— 설정 바꿈: 높음 —")을 끼워 비교가 되게 함.

### ③ 조종 미션
**미션 A — 낮음 vs 높음 대결판**
- 화면을 좌우 2칸으로: 왼쪽 "낮음(0.2)" 고정, 오른쪽 "높음(1.9)" 고정. 같은 시나리오 문장.
- 각 칸의 [3번 뽑기] 버튼 → 결과 3줄이 카드로 쌓임.
- 두 칸 모두 완료되면 판정 질문 등장: "어느 쪽이 더 **비슷비슷**했나요?" (왼쪽/오른쪽 선택 → 정답 '낮음' 쪽이면 ✔ "맞아요! 낮으면 흔들림 없이 고른 답이 나와요.")

**미션 B — 목적에 맞게 조종하기 (4문제, 교재 표 그대로)**
| 상황 | 정답 |
|---|---|
| 학교 안내문 쓰기 | 낮음~보통 |
| 과학 개념 설명하기 | 낮음~보통 |
| 동화 아이디어 떠올리기 | 보통~높음 |
| 별난 캐릭터 이름 짓기 | 보통~높음 |
- 상황 카드가 하나씩 나오고 [낮음~보통]/[보통~높음] 2버튼 선택. 정답 시 mint, 오답 시 coral + 한 줄 이유("안내문은 정확함이 중요해요"). 4문제 완료해야 [다음] 활성화.

## 4. 파이프라인/시각화 설계

15강 `.pipe/.pnode` 패턴 재사용. [문장 만들기] 클릭 시 4노드가 순서대로 점등(각 350ms):

```
1. 앞 문장을 읽어요        → "달나라 토끼 토토는 상자를 열자, 안에서…"
2. 다음 말 후보를 떠올려요  → 후보 5개 + 지금 확률 (편지 41% · 지도 32% · …)
3. 확률 룰렛을 돌려요      → 룰렛 연출: 후보 막대 위를 하이라이트가 좌→우로 훑다가 감속하며 멈춤 (~0.9s)
4. 문장이 완성돼요         → "…안에서 **말하는 우산**이 나왔습니다." (등급 배지: 🔥 새로움)
```
- 노드 3의 룰렛은 시각 연출일 뿐, 실제 선택은 클릭 순간 `pickWeighted()`로 확정(연출과 결과 분리 → 버그 방지).
- 뽑힌 단어에는 등급 태그를 붙임: 🧊 익숙함 / 🌱 조금 새로움 / ✨ 새로움 / 🔥 엉뚱함 — "높음 설정에서 🔥가 자주 나오네?"를 스스로 발견하게 유도.
- 확률 막대는 파이프라인 노드 2와 동일 데이터를 공유(한 소스, 두 뷰).

### 8-2. 핵심 로직 — 온도 소프트맥스와 가중 추첨

```js
// 슬라이더 0~100 → T 0.15~2.0 (지수 매핑으로 저온 구간을 넓게 체감시킴)
function sliderToT(v){ return Math.max(0.15, 0.15 * Math.pow(2.0/0.15, v/100)); }

// 소프트맥스: p_i ∝ exp( ln(base_i) / T )
// T→0.15: base 큰 후보(편지)가 거의 독식 / T→2.0: 다섯 후보가 비슷해짐
function probs(cands, T){
  const logits = cands.map(c => Math.log(c.base) / T);
  const m = Math.max(...logits);                      // overflow 방지
  const exps = logits.map(x => Math.exp(x - m));
  const sum = exps.reduce((a,b)=>a+b, 0);
  return exps.map(e => e / sum);
}
// 체감 예(S1): T=0.2 → 편지 87%·지도 13%·나머지 ≈0%
//             T=1.0 → 40/30/16/9/5%   T=2.0 → 28/24/18/15/13% (달빛 고래도 종종!)

function pickWeighted(cands, p){
  let r = Math.random(), acc = 0;
  for (let i = 0; i < p.length; i++){ acc += p[i]; if (r < acc) return i; }
  return p.length - 1;
}

// 한국어 조사 처리: 마지막 글자 받침 → 이/가
function josa(word, ga){ // ga=true: 이/가, false: 은/는 등 확장 여지
  const code = word.charCodeAt(word.length-1) - 0xAC00;
  const hasBatchim = code >= 0 && code % 28 !== 0;
  return hasBatchim ? '이' : '가';
}
```
