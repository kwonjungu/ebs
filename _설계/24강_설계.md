## 1. 학습 목표 연결 (교재 sec25~30)

| 교재 내용 | 체험 웹 대응 |
|---|---|
| 도윤이의 말만 못 알아듣는 AI 스피커 (sec25 생각 열기) | 사건 도입 스토리: 같은 명령을 두 아이가 말했을 때 결과가 갈리는 재현 연출 |
| 전체 정확도 82% vs 상황별 95%/40% — 평균에 가려진 편향 (sec26) | ② 사건 접수: 전체 점수만 먼저 보여 주고 '문제없어 보인다'에 투표시킨 뒤, ③에서 상황별 막대 차트를 열어 반전 |
| 편향은 사람 탓이 아니라 데이터의 부족·치우침 (sec26) | 원인 추리에서 '도윤이 말투 탓' 선택지를 명시적으로 기각, 데이터 접시 시각화 |
| 원인 가설: 데이터 부족/치우침/테스트 부족 (sec28 ④) | ③ 원인 추리 선택 인터랙션 |
| 해결안: 데이터 골고루 모으기·재테스트·대체 방법 (sec28 ⑤) | ③ 해결 시뮬레이션: '데이터 골고루 추가' 버튼 → 차트 개선 + 대체 방법 카드 |
| 사건 2 'AI 출입문'(90% vs 47%/53%) (sec28) | 보너스 사건(선택)으로 동일 흐름 반복 |
| 확인 문제 (sec29) | ④ 퀴즈 2문항 |

핵심 메시지: **"사람 탓이 아니라 데이터 문제"** — 수사 결말과 배지 문구에 반복.

## 3. 화면·상호작용 상세

### ① 탐정 준비
- 히어로: 칩 `AI 탐험대 · 중급 24강`, 제목 `⚖️ AI 편향 탐정 사무소`, 부제 `점수 하나에 숨은 비밀을 찾아내요.`
- 의뢰 카드(편지 봉투 연출): `[사건 의뢰서] 우리 교실 AI 스피커가 도윤이의 말만 자꾸 못 알아들어요. 도윤이는 또박또박 말했는데도요. 탐정님, 이유를 밝혀 주세요!`
- 탐정 약속 체크 3개(6절) → `사건 접수하기` 활성화.

### ② 사건 접수 — 전체 점수의 함정
- 사건 파일 카드: `사건 1 — 귀가 편식하는 AI 스피커` (돋보기 아이콘, 서류 폴더 감성).
- 대형 점수판(도넛 게이지 애니메이션): `전체 정확도 82%` + AI 회사 광고 문구: `"열 번 중 여덟 번은 알아들어요! 꽤 똑똑하죠?"`
- 1차 판단 투표: `이 AI, 문제가 있을까요?`
  - `😊 82%면 문제없어 보여요` / `🤔 뭔가 수상해요`
  - 어느 쪽을 눌러도 진행. 선택 결과는 ③ 반전 때 다시 소환: `아까 '문제없어 보여요'를 골랐다면, 지금은 어떤가요?`
- 제보 카드 슬라이드 인: `그런데 도윤이의 제보가 도착했어요 — "나는 분명히 말했는데, 왜 내 말만 자꾸 못 알아듣지?"` → `상황별 기록 열어 보기 🔍`

### ③ 수사와 해결 (핵심 미션 — 4장면 연속)

**장면 1. 증거 차트 열기 (반전)**
- `상황별 기록 열어 보기` 버튼 → 서류가 펼쳐지며 막대 차트 등장(막대가 왼쪽부터 차오르는 애니메이션):

| 상황 | 정확도 | 막대 색 |
|---|---|---|
| 조용한 교실 · 표준 말투 | 95% | mint |
| 어린이 목소리 | 55% | amber |
| 지역 말투 (도윤이) | 50% | coral |
| 시끄러운 교실 | 40% | coral |

- 점선 가로선으로 `전체 82%` 표시 → 캡션: `평균 점수 하나가 낮은 점수들을 가리고 있었어요!`
- 학생 조작: 낮은 막대(70% 미만)를 탭하면 `🚨 편향 의심!` 스탬프가 찍힘. 2개 이상 찍으면 다음으로.

**장면 2. 증거 문장 완성 (교재 sec28 ③ 서식)**
- 빈칸 채우기(드롭다운 3개):
  `이 AI의 전체 결과는 [82%로 좋아] 보여요. 하지만 [지역 말투 / 시끄러운 교실] 상황에서는 [50% / 40%]로 낮았어요. 그래서 이 AI는 [특정 말투·소음 환경]에서 편향이 의심돼요.`
- 올바르게 채우면 도장 애니메이션 `증거 확보!`

**장면 3. 원인 추리 (가설 세우기)**
- 질문: `왜 이런 일이 생겼을까요? 원인이라고 생각하는 카드를 모두 골라요.`
- 카드 5장(복수 선택):
  - `📉 데이터 부족 — 지역 말투·어린이 목소리 데이터가 너무 적었어요` ✔
  - `⚖️ 데이터 치우침 — 조용한 곳의 어른 목소리만 잔뜩 배웠어요` ✔
  - `🧪 테스트 부족 — 전체 점수만 보고 상황별로 확인하지 않았어요` ✔
  - `🗣️ 도윤이 말투가 이상해서` ✘ → 흔들림 애니메이션 + `아니에요! 편향은 사람 탓이 아니에요. AI가 그 말투를 충분히 배우지 못한 거예요.`
  - `😠 AI가 도윤이를 미워해서` ✘ → `AI는 미워하는 마음이 없어요. 배운 데이터가 문제예요.`
- ✔ 3개를 모두 고르면(✘는 골라도 감점 없음, 교정 피드백만) 다음으로.

**장면 4. 해결 시뮬레이션 — "데이터 골고루 추가"**
- 화면 구성: 왼쪽(위)에 **데이터 급식판**(교재 sec27 삽화 콘셉트) — 4칸 접시에 데이터 알갱이가 쌓여 있음. 오른쪽(아래)에 장면 1의 막대 차트 실시간 연동.
  - 시작 상태: 조용·표준 5,000개(수북) / 어린이 400개 / 지역 말투 300개 / 소음 환경 250개(거의 빈 접시 + `!` 표시)
- 조작: 부족한 접시마다 `➕ 데이터 골고루 추가` 버튼. 1회 탭 = +500개. 탭할 때마다:
  - 접시에 알갱이가 또르르 쌓이는 애니메이션
  - 해당 상황 막대가 계단식으로 상승 (8-2 수치표), 색이 coral→amber→mint로 변화
  - 전체 정확도 게이지도 함께 재계산되어 상승
- 목표: 모든 상황 85% 이상 → 팡파레 + 도넛 게이지 `전체 91%` + 메시지:
  `도윤이의 말투는 그대로인데 AI가 좋아졌어요. 범인은 사람이 아니라 데이터였어요!`
- 마무리 카드 — 안전장치 한 가지 더: `그래도 AI는 틀릴 수 있어요. AI가 못 알아들을 때를 위해 대체 방법을 골라요.` → 선택지: `🔘 버튼으로 입력하기` / `🔁 무조건 다시 말하게 하기` / `🚫 그 사람은 못 쓰게 하기` (정답: 버튼 입력 — 나머지는 교정 피드백).
- (선택) `보너스 사건 수사하기`: 사건 2 'AI 출입문' (전체 90% / 모자·두꺼운 안경 47% / 키 작은 경우 53%) — 같은 4장면 축약 반복. 건너뛰기 가능.

### ④ 수사 보고서 + 퀴즈 + 배지
- 수사 보고서 카드(서류 스탬프 감성):
  - `밝혀낸 것: 전체 82% 뒤에 40%가 숨어 있었어요.`
  - `원인: 데이터 부족 · 치우침 · 테스트 부족`
  - `해결: 부족한 상황의 데이터를 골고루 추가 + 상황별 재테스트 + 대체 방법 준비`
  - 핵심 문장(강조 박스): `편향은 사람 탓이 아니라 데이터 문제예요.`
- 퀴즈 2문항(7절) → `배지 받기 🏅` → 배지 화면.

## 4. 파이프라인/시각화 설계

**수사 파이프라인(진행 노드)** — ③ 상단에 15강 `.pnode` 패턴으로 고정:

```
[1 전체 점수 보기] → [2 상황별로 나누기] → [3 원인 추리] → [4 데이터 추가] → [5 다시 확인]
```
- 장면이 진행될 때마다 노드가 `on→ok`로 점등. "상황별로 나누어 확인"이 편향 발견의 절차임을 몸에 익히게 함.

**핵심 시각화 3종 (전부 바닐라 JS + CSS, 라이브러리 없음)**
1. **도넛 게이지**(전체 정확도): SVG `stroke-dasharray` 애니메이션. 82% → 시뮬레이션 후 91%.
2. **상황별 막대 차트**: div 막대의 `width` 트랜지션(가로 막대 — 모바일에서 라벨 읽기 유리). 점수 구간별 색: ≥85 mint / 60~84 amber / <60 coral. `전체 82%` 기준 점선 오버레이.
3. **데이터 급식판**: 4칸 grid, 칸마다 알갱이(작은 원) 개수 = 데이터량/250. 부족 칸에 `!` 배지. 버튼 탭 시 알갱이 추가 + 차트 연동.

## 5. 폴백·예외 처리

| 상황 | 처리 |
|---|---|
| 카메라·마이크·네트워크 의존 | 없음. 모든 결과표·차트 데이터는 JS 상수(가상 데이터) → 어떤 환경에서도 동일하게 동작 |
| 가상 데이터 오해 방지 | 차트 하단 상시 캡션: `이 숫자는 수업용으로 만든 가상 결과예요.` |
| 증거 문장 빈칸 오답 | 즉시 힌트: `차트에서 제일 낮은 막대를 다시 봐요.` 2회 오답 시 해당 막대 깜빡임 |
| 원인 추리에서 ✘만 고르는 경우 | 교정 피드백 후 ✔ 카드에 은은한 빛 효과로 유도 |
| 시뮬레이션 과다 탭 | 각 상황 90%에서 상한(더 눌러도 `이제 충분해요!`) — 무한 상승 방지, '데이터만 늘리면 100%'라는 오개념 차단 문구: `데이터를 더해도 100%는 안 돼요. 그래서 대체 방법이 필요해요.` |
| 보너스 사건 | 선택 사항 — 건너뛰어도 ④ 진행 가능 |
| 저사양 기기 | 애니메이션은 CSS transition 위주, `prefers-reduced-motion` 시 즉시 값 반영 |

## 6. 안전·안내 문구 (① 탐정 약속)

- ☑ `AI가 틀렸을 때 사람을 탓하지 않아요. AI가 무엇을 배웠는지 먼저 살펴봐요.`
- ☑ `전체 점수 하나만 믿지 않고, 누구에게·어떤 상황에서 틀리는지 나누어 확인해요.`
- ☑ `친구의 말투·모습을 놀리는 데 이 체험을 쓰지 않아요.`
- 하단 고정 안내: `이 체험의 AI 점수는 수업용 가상 데이터예요. 입력하거나 저장되는 정보는 없어요.`

## 7. 퀴즈 + 배지

**Q1.** AI 편향이 생기는 까닭으로 알맞은 것은 무엇인가요?
- ① AI가 특정 사람을 미워해서
- ② 그 사람의 말투가 이상해서
- ③ **AI가 배운 데이터가 부족하거나 한쪽으로 치우쳐서** ✔

**Q2.** 편향을 알아차리려면 무엇을 살펴봐야 하나요?
- ① 전체 정확도 점수 하나만 본다
- ② **누구에게, 어떤 상황에서 더 자주 틀리는지 나누어 본다** ✔
- ③ AI를 만든 회사 이름을 본다

**배지**: ⚖️ **공정 수사관** — 배지 화면 문구: `오늘 밝혀낸 것: 편향은 사람 탓이 아니라 데이터의 부족과 치우침 때문이에요. 상황별로 나누어 확인하고, 데이터를 골고루 모으고, 대체 방법을 준비해요.`

## 8. 기술 구현 노트

### 8-1. 사건 데이터 (JS 상수 — 실제 콘텐츠)

```js
const CASE1 = {
  name:'귀가 편식하는 AI 스피커', overall:82,
  situations:[
    { key:'quiet',  label:'조용한 교실 · 표준 말투', acc:95, data:5000, addable:false },
    { key:'child',  label:'어린이 목소리',           acc:55, data:400,  addable:true  },
    { key:'accent', label:'지역 말투 (도윤이)',       acc:50, data:300,  addable:true  },
    { key:'noise',  label:'시끄러운 교실',           acc:40, data:250,  addable:true  },
  ],
  // 데이터 추가 시 정확도 계단표: 탭 횟수(+500개)당 상승, 90% 상한
  accSteps:{ child:[55,70,81,88,90], accent:[50,66,79,87,90], noise:[40,58,73,84,89] },
  causes:{ correct:['lack','skew','test'], wrong:['personBlame','hate'] },
  fallbackAnswer:'button'   // 대체 방법 정답: 버튼 입력
};
const CASE2 = { // 보너스 사건
  name:'눈이 편식하는 AI 출입문', overall:90,
  situations:[
    { label:'평상시 모습', acc:96, addable:false },
    { label:'모자·두꺼운 안경 착용', acc:47, addable:true },
    { label:'키가 작은 경우', acc:53, addable:true },
  ]
};
```

### 8-2. 시뮬레이션 핵심 로직 의사코드

```
state.taps = { child:0, accent:0, noise:0 }

onAddData(key):
  if taps[key] >= 4: toast('이제 충분해요!'); return
  taps[key]++
  sit = CASE1.situations[key]
  sit.data += 500
  sit.acc = CASE1.accSteps[key][ taps[key] ]
  renderPlate(key)                       // 알갱이 +2 (500/250)
  animateBar(key, sit.acc)               // width 트랜지션 + 색 재계산
  overall = round( sum(sit.acc × weight) )   // weight: quiet .55 / child .15 / accent .15 / noise .15
  animateDonut(overall)
  if all(sit.acc >= 85): unlockNext()    // 팡파레 + '범인은 데이터였어요!' 카드

renderBarColor(acc): acc>=85 ? mint : acc>=60 ? amber : coral
```
- 가중 평균으로 전체 점수를 재계산해 "낮은 상황을 끌어올리면 평균도 오른다 + 그래도 평균만 봐선 몰랐다"를 수치로 일관되게 보여 줌 (시작값: .55×95 + .15×55 + .15×50 + .15×40 ≈ 82).

### 8-3. 차트·게이지 구현 메모
- 막대: `<div class="bar"><div class="fill" style="width:40%"></div><span>40%</span></div>`, `.fill { transition: width .6s ease, background .4s; }`
- 도넛: `<svg><circle r=54 stroke-dasharray="339" stroke-dashoffset=... />` — offset = 339×(1−acc/100), 숫자는 rAF 카운트업.
- 스탬프(`편향 의심!`·`증거 확보!`): `transform: scale(2)→1 + rotate(-8deg)` keyframe, 코럴/앰버 도장 스타일.
- 15강의 `go(n)`·진행 바·약속 체크·퀴즈·배지 코드 이식, 포인트 컬러 amber 치환.
