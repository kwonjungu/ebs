# 23강 체험 웹 설계서 — 📦 찰칵! 네모 박스 탐험: 웹캠 객체 탐지

- 경로: `/23/index.html` (단일 파일, GitHub Pages 정적 호스팅, 서버·API 키 없음)
- 포인트 컬러: **mint** (`--mint #34D399`, `--mint-light #D1FAE5`)
- 기술: **MediaPipe Tasks Vision (jsdelivr CDN) Object Detector — EfficientDet-Lite0** 로 웹캠 실시간 객체 탐지 (허용 CDN 규칙 준수)
- 카메라 사용 강 → 안전 문구·초상권 안내·시뮬레이션 폴백 필수

## 1. 학습 목표 연결 (교재 sec20~25)

| 교재 내용 | 체험 웹 대응 |
|---|---|
| 객체 탐지 = 여러 물체를 찾아 위치를 네모 박스로 표시 (sec21) | ② 단계: 웹캠 화면 위에 실시간 바운딩 박스+이름+`인식한 사물 수: n` 표시 — 생각 열기의 AI 편의점 장면 재현 |
| 이미지 분류 vs 객체 탐지 비교 (sec21) | ② 도입 토글: 같은 화면을 '분류 모드(한 줄 답)' ↔ '탐지 모드(박스들)'로 전환해 차이 체감 |

| 한계 살펴보기: 멀리/가림/어둡게/겹치기 (sec23) | ③ 미션: 4가지 상황 관찰 체크리스트 |
| 초상권·개인정보 보호 (sec23, 25) | ① 약속 + 상시 배너 `얼굴은 비추지 않아요` |
| 결과는 사람이 한 번 더 확인 (sec24) | ③ 마무리 메시지·④ 퀴즈·배지 문구 |

## 2. 단계 구성 (진행 바)

```
① 카메라 약속 → ② 네모 박스 체험 → ③ 한계 탐험 미션 → ④ 확인하고 배지 받기
```

| 단계 | 내용 | 완료 조건 |
|---|---|---|
| ① 카메라 약속 | 안전·초상권 약속 3개 체크 → 카메라 켜기(모델 로딩 병행) | 체크 3개 + 카메라 허용(불가 시 폴백 분기) |

## 3. 화면·상호작용 상세

### ① 카메라 약속
- 히어로: 칩 `AI 탐험대 · 중급 23강`, 제목 `📦 찰칵! 네모 박스 탐험`, 부제 `AI가 물건을 찾아 네모 박스로 짚어 줘요.`
- 준비 안내 카드: `책상 위에 컵, 책, 가위 같은 물건을 1~3개 준비해요. (엔트리 사물 인식처럼 3개까지 진하게 표시돼요)`
- 약속 체크 3개(6절 문구) → `카메라 켜고 시작하기` 활성화.
- 버튼을 누르면: (a) `getUserMedia` 권한 요청, (b) 동시에 MediaPipe 모델 다운로드 시작(로딩 스피너 + `AI 눈을 준비하고 있어요… 약 5초`).
- 실패/거부 시 폴백 안내 표시 후 `연습 모드로 시작하기` 버튼 (5절).

### ② 네모 박스 체험
- 중앙: 비디오(거울 반전 없음 — 박스 좌표 일치) + 그 위 `<canvas>` 오버레이.
  - 상위 3개: 굵은 민트 박스(3px) + 이름표(한글) + 신뢰도 `%`.
  - 4위 이하: 회색 점선 1px 박스, 이름 없음. 캡션: `엔트리 사물 인식처럼 3개까지만 진하게 표시해요.`
  - 우상단 계기판: `인식한 사물 수: n` (편의점 장면과 동일 문구, 숫자 변할 때 통통 튀는 애니메이션).
- 카메라 화면 바로 아래 상시 배너(민트라이트): `📷 화면에만 보이고 저장되지 않아요 · 얼굴은 비추지 말고 물건만 비춰요`
- **분류 vs 탐지 토글**(세그먼트 버튼):
  - `이미지 분류라면?` — 박스를 모두 숨기고 화면 아래 한 줄만: `"책상 사진입니다." (한마디로 끝!)`
  - `객체 탐지!` — 박스+이름+개수 복원: `"컵은 여기, 책은 여기!" (위치까지 짚어 줘요)`
- 관찰 노트(amber note): `물건을 슬쩍 옮겨 봐요. 네모 박스가 따라오나요? AI가 위치까지 찾고 있다는 증거예요.`

### ③ 한계 탐험 미션 — "AI 눈 시험하기"
- 같은 카메라 화면 유지, 아래 실험 체크리스트 4개 (교재 sec23 표 그대로):

| 실험 | 화면 안내 문구 | 체크 |
|---|---|---|
| 🚶 멀리 두기 | `물건을 카메라에서 멀리 옮겨 봐요. 박스가 사라지나요?` | 사물 수가 줄었어요 ○/✕ |
| ✋ 가리기 | `물건을 손으로 반쯤 가려 봐요.` | ○/✕ |
| 🌙 어둡게 하기 | `불을 끄거나 카메라를 그늘로 향해 봐요.` | ○/✕ |
| 🧱 겹쳐 놓기 | `물건 두 개를 겹쳐 놓아 봐요.` | ○/✕ |

- 각 실험 행은 접이식: 펼치면 안내 문구 + ○/✕ 버튼. 실험 중 실시간 `인식한 사물 수`가 함께 보여 변화를 바로 확인.
- 4개 체크 완료 시 발견 카드: `AI 눈도 만능이 아니에요. 멀거나 가려지면 놓쳐요. 그래서 중요한 일은 사람이 한 번 더 확인해요. (자율주행차가 "핸들을 잡아 주세요!" 하는 까닭이에요)`

### ④ 퀴즈 + 배지
- 퀴즈 2문항(7절) → 배지. 배지 화면에서 카메라 스트림 완전 종료(`track.stop()`) + `카메라를 껐어요` 표시.

## 4. 파이프라인/시각화 설계

교재 sec22의 3단계를 15강 `.pnode` 패턴으로, ② 화면 카메라 아래 배치:

```
[1 비추기 📷 카메라가 물건을 비춰요]
→ [2 위치 찾기 🔲 AI가 자리를 찾아 네모 박스를 그려요]
→ [3 이름·개수 표시 🏷️ "컵" … 인식한 사물 수: 2]
```

- 탐지 루프가 돌 때: 1번 노드는 항상 `on`, 탐지 결과 ≥1이면 2·3번이 순서대로 `ok` 점등(150ms 간격), 결과 0이면 2번이 `fail` + `아직 물건을 못 찾았어요`. 3번 노드 텍스트는 실시간 갱신(대표 이름 + 개수).
- 바운딩 박스 = 핵심 시각화 그 자체: 박스 모서리 등장 애니메이션(scale 1.06→1.0), 이름표는 박스 위 모서리에 민트 라벨.

## 5. 폴백·예외 처리 (연습 모드: 직접 박스 그리기 게임)

| 상황 | 처리 |
|---|---|
| 카메라 거부/미지원(iframe·구형 브라우저) | **연습 모드**로 자동 분기(아래) |
| 모델 다운로드 실패(오프라인) | 15초 타임아웃 → `AI 눈을 불러오지 못했어요. 연습 모드로 체험해요.` → 연습 모드 |
| WebGL 미지원 | delegate `'CPU'`로 재시도 → 실패 시 연습 모드 |
| 탐지 루프 저성능 기기 | `detectForVideo`를 rAF마다가 아니라 최소 100ms 간격 스로틀 |
| 페이지 이탈/단계 이동 | `visibilitychange`·`go(4)`에서 스트림 정지 |

**연습 모드(폴백) — "내가 AI가 되어 박스 그리기"**
- 소재: 책상 위 물건 장면을 **inline SVG**로 내장(외부 이미지 불필요): 컵 ☕, 책 📕, 가위 ✂️, 사과 🍎 가 놓인 책상 일러스트.
- 놀이 방법: `AI가 되어 물건마다 네모 박스를 그려 봐요!` — 터치 드래그로 사각형 그리기(최대 4개, 잘못 그리면 길게 눌러 삭제).
- `AI 결과와 비교하기` 버튼 → 미리 정의된 정답 박스(민트)가 겹쳐 표시 + IoU(겹침 비율) 간단 채점: 겹침 50% 이상이면 `찾았어요!` 뱃지. 결과 문구: `내 박스 3개 = AI 박스 3개! 객체 탐지는 이렇게 '어디에'까지 표시하는 일이에요.`
- ③ 한계 미션 폴백: 같은 SVG 장면의 변형 3컷(멀리=물건 축소, 가림=손 그림 겹침, 어둡게=어두운 필터)을 보여 주고 `AI가 찾을 수 있을까요?` 예상 ○/✕ → 정답 공개(전부 '놓치기 쉬움') 방식으로 체크리스트 대체.

## 6. 안전 문구 (① 약속 + 상시 배너)

- ☑ `카메라 화면은 화면에만 보이고 저장되지 않아요.`
- ☑ `얼굴은 비추지 않아요. 책상 위 물건만 비춰요. 친구 얼굴이 나온 사진·영상은 함부로 찍거나 올리지 않아요. (초상권 보호)`
- ☑ `AI는 물건을 놓치거나 헷갈릴 수 있어요. 결과는 내가 한 번 더 확인해요.`
- ② ③ 화면 상시 배너: `📷 화면에만 보이고 저장되지 않아요 · 얼굴은 비추지 말고 물건만 비춰요`
- 사람(person)이 탐지되면 박스를 그리지 않고 상단에 안내 토스트: `사람이 보여요! 얼굴은 비추지 않기로 약속했어요. 카메라를 물건 쪽으로 돌려요.` (person 클래스는 표시 제외 목록으로 처리)

## 7. 퀴즈 + 배지

**Q1.** 사진 속 여러 물체를 찾아 각각의 위치를 네모 박스로 표시하는 기술은 무엇인가요?
- ① 이미지 분류 (해설: 분류는 사진 전체가 무엇인지 한 가지로만 말해요)
- ② **객체 탐지** ✔
- ③ 바코드 스캔

**Q2.** AI 카메라의 탐지 결과를 대하는 태도로 알맞은 것은 무엇인가요?
- ① 네모 박스가 그려졌으면 항상 맞으니까 확인하지 않는다

**배지**: 📦 **네모 박스 탐험가** — 배지 화면 문구: `오늘 배운 것: 비추기 → 위치 찾기 → 이름·개수 표시. AI는 멀거나 가려지면 놓칠 수 있어요. 마지막 확인과 초상권 보호는 사람의 몫이에요.`

## 8. 기술 구현 노트

### 8-1. MediaPipe 초기화 (핵심 코드 골격)

```js
import { ObjectDetector, FilesetResolver } from
  "https://cdn.jsdelivr.net/npm/@mediapipe/tasks-vision@0.10.14";

const vision = await FilesetResolver.forVisionTasks(
  "https://cdn.jsdelivr.net/npm/@mediapipe/tasks-vision@0.10.14/wasm");
detector = await ObjectDetector.createFromOptions(vision, {
  baseOptions: { modelAssetPath:
    "https://storage.googleapis.com/mediapipe-models/object_detector/efficientdet_lite0/float16/1/efficientdet_lite0.tflite",
    delegate: "GPU" },              // 실패 시 "CPU" 재시도
  runningMode: "VIDEO",
  scoreThreshold: 0.4,
  maxResults: 8                     // 8개 받아 상위 3개만 강조
});
```
- `<script type="module">` 사용. 모델(≈4.5MB)은 MediaPipe 공식 스토리지에서 로드(모델 파일은 CDN 패키지에 없음 — MediaPipe 구성요소로 허용 범위, 별도 API 키 없음).

### 8-2. 탐지 루프 의사코드

```
loop(now):
  if (video.currentTime != lastTime && now-lastRun > 100ms):
    res = detector.detectForVideo(video, now)
    dets = res.detections
      .filter(d => KO_LABEL[d.categories[0].categoryName] != null)   // 미등록 라벨 제외
      .filter(d => d.categories[0].categoryName != 'person')          // 사람 박스 미표시
    if (containsPerson(res)) showFaceToast()
    dets.sort(by score desc)
    top3 = dets.slice(0,3); rest = dets.slice(3)
    drawCanvas(top3: 민트 굵은 박스+한글라벨+%, rest: 회색 점선)
    countEl.textContent = '인식한 사물 수: ' + top3.length
    updatePipeline(top3.length)
  requestAnimationFrame(loop)
```
- 캔버스는 비디오 실표시 크기에 맞춰 `resize` 시 재계산(박스 좌표 = boundingBox * scale).

### 8-3. 한글 라벨 매핑 (COCO 80종 중 교실에서 나올 것 위주)

```js
const KO_LABEL = { cup:'컵', bottle:'병', book:'책', scissors:'가위',
  'cell phone':'휴대폰', keyboard:'키보드', mouse:'마우스', laptop:'노트북',
  apple:'사과', banana:'바나나', orange:'오렌지', sandwich:'샌드위치',
  backpack:'가방', umbrella:'우산', chair:'의자', tv:'모니터',
  clock:'시계', spoon:'숟가락', fork:'포크', bowl:'그릇', vase:'꽃병',
  'teddy bear':'인형', 'sports ball':'공', remote:'리모컨', person:'사람' };
// 매핑에 없는 라벨은 표시하지 않음(혼란 방지). person은 매핑만 있고 표시 제외.
```

### 8-4. 폴백 박스 그리기 채점 의사코드

```
정답 데이터: ANSWERS = [ {label:'컵', x:60,y:120,w:90,h:110},
  {label:'책', x:200,y:150,w:150,h:90}, {label:'가위', x:400,y:130,w:120,h:80} ]  // SVG 좌표계
드래그: pointerdown 시작점 → pointermove로 rect 미리보기 → pointerup 확정(최소 20×20)
채점: 각 정답 박스에 대해 학생 박스 중 IoU 최대값 계산
  IoU = 교집합넓이 / 합집합넓이, ≥0.5 → '찾았어요!' / <0.5 → '조금 아쉬워요, AI 박스와 비교해 봐요'
```

### 8-5. 기타
- 15강의 진행 바·약속 체크·퀴즈·배지 코드 이식, 포인트 컬러만 mint로 치환.
- 카메라: `getUserMedia({ video:{ facingMode:'environment', width:{ideal:640} }, audio:false })` — 태블릿 후면 카메라 우선(책상을 비추는 용도), 실패 시 기본 카메라.
- 종료 처리: 배지 화면 진입·`pagehide` 시 `stream.getTracks().forEach(t=>t.stop())`.
