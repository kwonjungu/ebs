# 22강 체험 웹 설계서 — 🕵️ 팩트체크 탐정단: AI의 자신 있는 말, 진짜일까?

- 경로: `/22/index.html` (단일 파일, GitHub Pages 정적 호스팅, 서버·API 키 없음)
- 포인트 컬러: **coral** (`--coral #F87171`, `--coral-light #FEE2E2`)
- 카메라·마이크: 사용 안 함 (권한 요청 없음 → 폴백 부담 최소)
- 레퍼런스: 15강 구조(진행 바·카드·파이프라인 노드·퀴즈·배지) 그대로 재사용

## 1. 학습 목표 연결 (교재 sec16~20)

| 교재 내용 | 체험 웹 대응 |
|---|---|
| 할루시네이션 개념: 사실과 다른 내용을 그럴듯한 문장으로 만들어 내는 현상 (sec16) | ② 단계에서 AI가 '자신 있는 말투'로 말하는 상식 카드를 직감으로 예상해 보고, 자신 있는 말투 ≠ 사실임을 몸으로 발견 |
| 팩트 체크 5단계: 표시→찾기→비교→나누기→사용 (sec17) | ③ 단계 카드마다 5단계 파이프라인 노드가 순서대로 켜지며 진행 |
| 상식 카드 5장 판정 활동 (sec18): 달 착륙·화성 위성·퀴리 노벨상·만리장성·달 크레이터 | ③ 단계 미션의 카드 5장을 교재와 동일한 소재·동일한 판정 기준(확인됨/틀림/일부 다름/근거 없음)으로 구성 |
| "근거 자료를 함께 주고 비교" + 마지막 확인은 사람 (sec17, 20) | 근거 자료 카드(NASA·노벨상 누리집 요약)를 열어 나란히 비교하는 상호작용, 배지 화면 핵심 정리 |
| 확인 문제 (sec19) | ④ 퀴즈 2문항 |

핵심 발견 유도: **"내 직감(예상) 적중률"과 "근거 확인 후 판정 정확도"를 숫자로 비교**해서, '그럴듯해 보여도 확인해야 안다'를 스스로 깨닫게 한다.

## 2. 단계 구성 (진행 바)

```
① 탐정 준비 → ② 직감 예상 → ③ 팩트 체크 수사 → ④ 확인하고 배지 받기
```

| 단계 | 내용 | 완료 조건 |
|---|---|---|
| ① 탐정 준비 | 탐정 약속 3개 체크 + 할루시네이션 한 줄 소개 | 체크 3개 모두 |
| ② 직감 예상 | 상식 카드 5장을 보고 '참일 것 같아요/거짓일 것 같아요' 예상 투표 | 5장 모두 예상 |
| ③ 팩트 체크 수사 | 카드마다 5단계(표시→찾기→비교→나누기→사용) 진행, 근거 자료 열어 판정 | 5장 모두 판정 |
| ④ 확인하고 배지 받기 | 직감 vs 팩트체크 결과 비교 리포트 → 퀴즈 2문항 → 배지 | 퀴즈 2문항 정답 |

## 3. 화면·상호작용 상세

### ① 탐정 준비
- 히어로: 칩 `AI 탐험대 · 중급 22강`, 제목 `🕵️ 팩트체크 탐정단`, 부제 `AI가 자신 있게 말해도, 사실인지는 탐정이 확인해요.`
- 도입 카드(코럴 배경 말풍선): AI 캐릭터가 자신 있는 말투로 말함 —
  > "제가 알려 드릴게요! 아주 정확한 상식 5가지예요. 걱정 말고 믿으세요! 😎"
  아래에 작은 안내: `이렇게 자신 있게 말해도, 전부 사실일까요? 오늘은 우리가 탐정이 되어 확인해요.`
- 탐정 약속 체크 3개 (아래 6절 문구) → 모두 체크 시 `수사 시작하기` 버튼 활성화.

### ② 직감 예상 (자신 있는 말투 ≠ 사실 체험)
- 카드 5장이 한 장씩 넘어가는 방식(`카드 1/5` 표시). 각 카드:
  - AI 말풍선(자신 있는 말투로 문장 제시, 예: "확실해요! 아폴로 11호에서 달을 가장 먼저 밟은 사람은 버즈 올드린이에요.")
  - 버튼 2개: `⭕ 참일 것 같아요` / `❌ 거짓일 것 같아요` (44px 이상)
  - 누르면 즉시 다음 카드로. 정답은 아직 알려 주지 않음 — `예상 저장! 진짜인지는 ③에서 수사해요.`
- 5장 완료 시: `직감 예상 완료! 이제 근거 자료로 확인해 볼까요?` + `팩트 체크 수사 시작 →`

### ③ 팩트 체크 수사 (핵심 미션)
카드 1장 = 미니 수사 1건. 화면 상단에 5단계 파이프라인(4절), 그 아래 수사 카드.

카드 안 진행 순서(버튼을 누를 때마다 파이프라인 노드가 켜짐):

1. **표시하기** — AI 문장 속에서 숫자·이름·과학 사실 단어가 **밑줄 칩**으로 표시됨. 학생이 칩을 탭하면 형광펜 효과(코럴 하이라이트). 확인 필요 칩을 1개 이상 탭하면 다음 버튼 활성화. 안내: `숫자·이름·과학 사실을 탭해서 표시해요.`
2. **찾기** — `🔍 근거 자료 열기` 버튼. 누르면 근거 자료 카드가 봉투 열리는 애니메이션(CSS)으로 등장. 출처 배지(`NASA 공식 자료 요약` 등) + 요약 2~3문장.
3. **비교하기** — AI 답변과 근거 자료가 **좌우 2단(모바일은 상하)** 으로 나란히 배치. 다른 부분이 자동으로 코럴/민트 하이라이트. 안내: `두 글을 나란히 놓고 다른 곳을 찾아요.`
4. **나누기(판정)** — 버튼 4개: `확인됨` (mint) / `틀림` (coral) / `일부 다름` (amber) / `근거 없음` (gray). 선택 즉시 정답 여부 + 해설 1~2문장 표시. 오답이면 해설 읽고 재선택 가능(첫 선택만 점수 기록).
5. **사용하기** — 판정 결과에 따라 자동 분류 애니메이션: `확인됨` 카드는 `📗 발표 자료에 넣어요` 폴더로, 나머지는 `🗑️ 빼거나 바르게 고쳐요` 폴더로 날아감. 고친 문장(정답 문장)을 한 줄 보여줌.

- 카드 하단에 `내 직감: 참일 것 같다 → 실제: 틀림 😲` 식으로 ②의 예상과 즉시 대조.
- 5건 완료 시 `수사 결과 보고서 →` 버튼.

### ④ 수사 결과 보고서 + 퀴즈 + 배지
- 보고서 카드:
  - `내 직감 적중: n/5` vs `근거 확인 후 판정: m/5` 막대 2개 비교.
  - 핵심 메시지: `자신 있는 말투는 사실의 증거가 아니에요. 근거 자료와 비교한 다음에만 사용해요.`
  - AI가 5장 중 4장을 틀렸다는 요약: `AI는 거짓말쟁이가 아니에요. 근거가 부족한 곳을 그럴듯한 말로 채운 거예요. 그래서 마지막 확인은 사람이 해요.`
- 퀴즈 2문항(7절) → 모두 정답 시 `배지 받기 🏅` 활성화 → 배지 화면.

## 4. 파이프라인/시각화 설계

15강 `.pipe / .pnode` 패턴 재사용. 단계 이름은 교재 sec17의 5단계 그대로:

```
[1 표시하기] → [2 찾기] → [3 비교하기] → [4 나누기] → [5 사용하기]
```

- 상태: 대기(회색) → 진행 중 `on`(코럴 테두리) → 완료 `ok`(민트). '틀림/근거 없음' 판정 시 4번 노드는 `fail`(코럴)로 표시하되 "잘 걸러냈어요!"라는 긍정 메시지(걸러낸 것이 성공).
- 카드가 바뀔 때마다 노드 리셋. 5장 수사 동안 같은 5단계가 반복되며 절차가 손에 익도록 한다.
- 보조 시각화: ⑤ 사용하기의 두 폴더(`📗 사용` / `🗑️ 제외·수정`) — 판정된 카드가 folder로 이동하는 transform 애니메이션.

## 5. 폴백·예외 처리

| 상황 | 처리 |
|---|---|
| 카메라·마이크 미사용 | 권한 이슈 자체가 없음. 모든 데이터는 JS 상수로 내장 → 오프라인 캐시 후에도 동작 |
| 외부 링크(NASA 등) | 실제 사이트로 이동시키지 않음. 근거 자료는 **페이지 내 요약 카드**로 내장(수업 흐름 유지 + 네트워크 무관). 출처명만 배지로 표기 |
| 판정 오답 반복 | 2회 오답 시 힌트 자동 표시: `근거 자료의 밑줄 부분을 다시 읽어 봐요.` 3회째는 정답 공개 + 해설(체험이 막히지 않게) |
| 중간 이탈 | 진행 상태를 메모리에만 유지(저장 없음). 새로고침 시 처음부터 — 배지 화면에 `처음부터 다시` 버튼 |
| 좁은 화면 | 비교하기 2단 배치는 480px 이하에서 상하 배치로 전환 |

## 6. 안전·안내 문구 (① 단계 탐정 약속)

- ☑ `AI가 자신 있게 말해도 사실이 아닐 수 있어요. 나는 근거를 확인하고 판단해요.`
- ☑ `숫자·이름·연도·과학 사실은 꼭 근거 자료와 비교해요.`
- ☑ `확인되지 않은 정보는 숙제나 발표에 쓰지 않아요.`
- 하단 고정 안내(footer 위): `이 체험의 AI 답변은 학습용으로 만든 예시예요. 입력하거나 저장되는 정보는 없어요.`

## 7. 퀴즈 + 배지

**Q1.** AI가 사실과 다른 내용을 그럴듯한 문장으로 만들어 내는 현상은 무엇인가요?
- ① AI가 일부러 하는 거짓말 (오답 해설: 속이려는 게 아니라 근거가 부족한 곳을 그럴듯하게 채운 거예요)
- ② **할루시네이션** ✔
- ③ AI가 모를 때 "모릅니다"라고 말하는 것

**Q2.** AI 답변을 가장 안전하게 다시 확인하는 방법은 무엇인가요?
- ① "이거 맞아?"라고 다시 묻고 그 답을 믿는다
- ② **근거 자료를 함께 주고, 자료와 비교해 달라고 한다** ✔
- ③ 자신 있게 말한 답이면 그냥 믿는다

**배지**: 🕵️ **팩트체크 탐정** — 배지 화면 문구: `오늘 배운 것: 표시 → 찾기 → 비교 → 나누기 → 사용. 자신 있는 말투보다 근거 자료를 믿어요. 마지막 확인은 사람이 해요.`

### 8-2. 핵심 로직 의사코드

```
상태: guesses[5](②의 참/거짓 예상), verdicts[5](③의 첫 판정), pipeStep(1~5)
```
