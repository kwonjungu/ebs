# 18강 QR 체험 웹 설계서 — 🎧 AI DJ! 내 기분을 읽고 노래를 골라요

- 경로: `/18/index.html` (단일 파일, GitHub Pages 정적 호스팅, 서버·API 키 없음)
- 포인트 컬러: **coral** (`--coral: #F87171`, `--coral-light: #FEE2E2`)
- 페이지 제목: `AI DJ! 내 기분을 읽고 노래를 골라요 — 중급 18강`
- 히어로: 칩 "AI 탐험대 · 중급 18강" / h1 "🎧 AI DJ! 내 기분을 읽고 노래를 골라요" / 서브 "일기 속 단어를 단서로 감정을 추정하고, 어울리는 음악을 추천해요. 추천은 정답이 아니라 예상이에요."
- 카메라·마이크 사용 안 함. 소리는 Web Audio 멜로디(선택 재생)만.

## 1. 학습 목표 연결 (교재 코너 대응)

| 체험 웹 단계 | 교재 코너 | 학습 목표 |
|---|---|---|
| ② 감정 단서 찾기 | 생각 열기(마루의 발표 일기), AI 탐험 미션(감정 단어 동그라미), 직접 적용해 보기(문장→단서→범주) | 글 속 감정 단서 단어를 찾고 감정 범주로 분류하는 과정 이해 |
| ② 비교표 연출 | 개념 쏙쏙 「마음을 읽는다 VS 문장으로 추정한다」 비교표 | AI는 속마음이 아니라 표현(단어)을 보고 추정함을 이해 |
| ③ AI DJ 추천 | 활용 및 적용 「3단계 흐름표」(일기 읽기→감정 분류→음악 추천), 감정별 추천 분위기 표 | 분류 결과가 추천으로 연결되는 흐름 이해 |
| ③ 피드백 체험 | 토론 「추천, 그대로 믿어도 될까?」, 생각하기(슬플 때 취향), 활용 가이드 1·2번 | 추천은 '예상'이며, 안 맞으면 피드백으로 고쳐 쓰는 태도 |
| ① 안전 약속 / ④ 퀴즈 | 활용 가이드 3번(개인정보 없이 짧게), AI 탐험가 확인 문제 | 입력 시 개인정보 주의, 배운 내용 확인 |

## 3. 화면·상호작용 상세

### ① 준비 — 안전 약속
- 체크 3개:
  1. "기분 일기에는 **이름·학교·전화번호 같은 개인정보를 적지 않아요.** 기분만 짧게 적어요."
  2. "내가 쓴 일기는 **이 화면에만 보이고 어디에도 저장·전송되지 않아요.**"
  3. "AI 추천은 **정답이 아니라 예상**이에요. 내 마음과 다를 수 있다는 걸 기억해요."
- 도입 카드: 마루의 발표 일기(생각 열기) 요약 + "AI DJ는 마루의 어떤 단어를 보고 '밝고 힘나는 음악'을 골랐을까요? 직접 AI DJ가 되어 확인해요!"

### ② 감정 단서 찾기 — "AI는 마음이 아니라 단어를 봐요"
- 화면 구성:
  1. **일기 입력부** — 두 가지 방법(탭 또는 병렬):
     - **예시 일기 카드 4장** (교재 '직접 적용해 보기' 문장): "시험을 잘 봐 뿌듯했다." / "비가 와서 축구를 못하게 되어 속상했다." / "새치기를 당해서 화가 났다." / "내일 발표가 있어 너무 떨린다." + 보너스 카드 "마루의 발표 일기"(감정이 섞인 긴 일기 — ③의 복합 감정 체험 씨앗).
     - **직접 쓰기** `textarea` (placeholder: "오늘 기분을 한두 문장으로 적어 봐요. 이름은 빼고요!", 최대 200자).
  2. 「🔍 감정 단서 찾기」 버튼 → **하이라이트 연출**: 일기 문장이 카드에 다시 표시되며 감정 사전에 있는 단어에 형광펜(감정별 색: 기쁨=amber, 슬픔=sky, 화남=coral, 불안=purple)이 왼쪽부터 차례로 칠해짐(120ms 간격).
  3. **감정 점수 막대 4개**: 기쁨 😊 / 슬픔 😢 / 화남 😠 / 불안 😰 — 단서 개수만큼 막대가 차오르고, 최고 막대에 왕관 👑 + "이 일기는 **불안/긴장**에 가까워 보여요" 라벨.
  4. 단서가 2개 감정 이상이면(마루 일기 등): "감정이 **섞여** 있어요! 떨렸다(불안)와 좋아졌다(기쁨)가 함께 있네요" 배지 — 복합 감정 인지.
- 관찰 노트(`.note`): "🔍 AI는 마루의 **속마음을 읽은 게 아니에요.** '떨리는', '기분이 좋아졌다' 같은 **단어**를 보고 추정했을 뿐이에요. 그래서 틀릴 수도 있어요."
- 미니 비교 카드(교재 비교표 축약): [흔한 오해] "AI가 마음을 읽는다" ↔ [실제] "문장 속 단어를 보고 비슷한 예시와 비교해 추정한다" — 탭하면 뒤집히는 플립 카드 2장.
- 예시 카드 2장 이상(또는 직접 쓰기 1회) 분석하면 「다음」 강조.

### ③ AI DJ 미션 — "추천은 예상! 내 마음과 맞는지 확인해요"
- 화면 구성:
  1. ②에서 분석한 일기와 감정 결과가 이어져 표시(새로 고르기도 가능).
  2. **3단계 파이프라인** (4절) 점등 후 **추천 카드** 등장: 감정별 음악 분위기 카드 —
     - 기쁨 😊 → "🌞 밝고 신나는 음악" / 슬픔 😢 → "🌙 잔잔하고 위로가 되는 음악" / 화남 😠 → "🌊 차분하게 가라앉히는 음악" / 불안 😰 → "🍃 느리고 편안한 음악"
     - 카드에 추천 **이유 문장** 필수 표기: "'떨린다'는 단어가 보여서, 마음이 편안해지는 음악을 골랐어요." — 추천 근거 투명화(토론 '만든 사람의 약속' 연결).
  3. 「▶ 분위기 들어보기」 버튼: Web Audio 오실레이터로 감정별 8초 멜로디 재생(기쁨=빠른 장조, 슬픔=느린 단조, 화남→차분=낮은 음 느린 템포, 불안→편안=부드러운 저음 아르페지오). 15강 `playMusic` 패턴 재사용. 재생 중 스피커 이모지 바운스.
  4. **피드백 코너** ★핵심: 추천 카드 아래 큰 버튼 2개 —
     - 「👍 내 마음과 맞아요」 → "좋아요! 그래도 추천은 예상이니 늘 확인하는 거예요 😊"
     - 「👎 내 마음과 달라요」 → **조정 패널** 등장: "그럼 지금 어떤 음악이 듣고 싶어요?" 분위기 카드 4장 중 직접 선택 → 추천 교체 + "피드백 고마워요! 다음부턴 이런 기분일 때 이 분위기를 먼저 추천할게요" → **취향 메모리 시각화**: 화면 위 '나의 취향 노트' 칩에 기록(예: "슬플 때 → 신나는 음악 🌞"). 이후 같은 감정 일기를 다시 분석하면 조정된 추천이 먼저 나옴 — "피드백이 추천을 바꾼다"를 체감.
  5. **미션 카드 2개**:
     - 미션1 "일기 2개 이상으로 추천을 받아 봐요" (자동 체크)
     - 미션2 "👎 버튼으로 추천을 한 번 바꿔 봐요 — 추천은 정답이 아니니까요!" (조정 1회 시 체크)
  6. 미션 완료 시 "🎉 이제 알겠죠? 추천은 **정답이 아니라 예상**이에요. 확인하고 고르는 건 나!" + 「다음: 확인 문제 →」.
- 탐험 실험 노트(`.note`): "🧪 '좋았다'라고만 쓰고 '반어법'인 상황을 상상해 봐요(사실은 속상한 날). AI가 속을까요? 단어만 보는 AI의 한계를 알 수 있어요. 감정 단어가 하나도 없는 일기('오늘 급식은 카레였다')도 넣어 봐요!"

### ④ 퀴즈 + 배지
- 2문항 정답 시 배지. 배지 카드 요약문: "오늘 배운 것: 일기 읽기 → 감정 분류(단어 단서) → 음악 추천. AI는 마음이 아니라 표현을 봐요. 추천은 예상이니 내가 확인하고, 안 맞으면 피드백해요."

## 4. 파이프라인 / 시각화 설계

③단계 추천 시 15강 `.pnode` 3노드 순차 점등(교재 3단계 흐름표 그대로):

```
[1] 일기 읽기  → "'내일 발표가 있어 너무 떨린다.'" (입력 문장 표시, on)
[2] 감정 분류  → "단서: 떨린다 → 불안/긴장 😰 (기쁨0·슬픔0·화남0·불안1)" (ok)
[3] 음악 추천  → "🍃 느리고 편안한 음악" (ok; 피드백 조정 시 "🌞 (내 피드백 반영)" 로 갱신)
```
- 감정 단어가 없는 일기 입력 시: [2]가 fail — "단서를 못 찾았어요. 감정을 나타내는 단어가 없나 봐요" → [3] "추천 어려움 — 기분을 나타내는 단어를 넣어 볼까요?" (AI의 한계를 파이프라인 상태로 보여줌).
- 복합 감정 시 [2]에 상위 2개 감정 병기: "불안1 + 기쁨2 → 기쁨이 조금 더 커요".
- ②의 감정 점수 막대(4색 가로 막대 + 왕관)가 이 강의 대표 시각화 — '분류=점수 비교'임을 직관화.

## 8. 기술 구현 노트

- **의존성 없음**: 외부 CDN 0개, 바닐라 JS. Web Audio는 15강 `playMusic` 오실레이터 패턴 확장.
- **감정 키워드 사전과 핵심 로직 의사코드**:

```js
const LEXICON = {
  joy:   { label:'기쁨/신남', emo:'😊', color:'amber',
           words:['뿌듯','기뻤','기쁘','신나','신났','좋았','좋아졌','행복','즐거','웃었','박수','칭찬','설레','최고'] },
  sad:   { label:'슬픔/우울', emo:'😢', color:'sky',
           words:['속상','슬펐','슬프','우울','눈물','아쉬','서운','외로','시무룩','울었'] },
  angry: { label:'화남/스트레스', emo:'😠', color:'coral',
           words:['화가','화났','짜증','억울','분했','약올','열받','싫었'] },
  anx:   { label:'불안/긴장', emo:'😰', color:'purple',
           words:['떨리','떨렸','긴장','걱정','불안','무서','두려','조마조마','초조'] },
};
const MUSIC = { joy:'🌞 밝고 신나는 음악', sad:'🌙 잔잔하고 위로가 되는 음악',
                angry:'🌊 차분하게 가라앉히는 음악', anx:'🍃 느리고 편안한 음악' };
let pref = {};                                  // 취향 노트 (메모리만, 저장 안 함)

function analyze(text){
  const hits = [];                              // [{word, cat, index}]
  for(const cat in LEXICON)
    for(const w of LEXICON[cat].words){
      let i = -1;
      while((i = text.indexOf(w, i+1)) !== -1) hits.push({word:w, cat, index:i});
    }
  const scores = countBy(hits, 'cat');          // 감정 막대 그래프 데이터
  const top = argmax(scores);                   // 동점이면 먼저 나온 단서의 감정
  return { hits, scores, top, mixed: Object.keys(scores).length >= 2 };
}
function highlight(text, hits){                 // 겹침 방지: index 정렬 후 <mark class=cat> 감싸기
  /* hits를 index 오름차순 정렬, 순서대로 형광펜 span 삽입 (120ms 간격 클래스 추가) */ }
function recommend(top){                        // 피드백 반영이 핵심
  const mood = pref[top] || top;                // 조정된 취향이 있으면 우선
  return { card: MUSIC[mood], adjusted: mood !== top };
}
function feedback(top, chosenMood){             // 👎 → 분위기 직접 선택
  pref[top] = chosenMood;                       // "슬플 때 → 신나는 음악" 기록
  renderPrefNote(); rerunPipeline();            // 파이프라인 [3] 갱신 연출
}
// 멜로디: MELODY[mood] = {notes:[Hz…], tempo, type:'triangle'|'sine'}
//   joy=[523,659,784,1046…] 320ms / sad=[440,392,330…] 640ms / angry→calm=[262,247,220] 700ms / anx→relax=[330,262,196] 800ms
//   재생 8초 후 자동 정지, 버튼 토글. 첫 탭에서 audioCtx.resume() (iOS)
// 개인정보 가드: /01[016789]-?\d{3,4}-?\d{4}|\d{7,}/ 매치 시 분석 차단 + 마스킹 안내
```
