## 1. 학습 목표 연결 (교재 코너 대응)

| 체험 웹 단계 | 교재 코너 | 학습 목표 |
|---|---|---|
| ② 탐지 vs 식별 | 개념 쏙쏙 「비교표 – 얼굴 탐지 VS 얼굴 식별」, 문항 6번(탐지/식별 고르기) | 얼굴이 '누구인지'(식별)가 아니라 '어디에 있는지'(탐지)를 찾는 차이 이해 |
| ③ 고양이 귀 AR | 생각 열기(마루 필터 앱), 개념 쏙쏙 「AR? 그게 뭐야?」, 용어 쏙쏙(특징점·X좌표·Y좌표) | AR 개념, 특징점이 X·Y 좌표 숫자로 바뀌는 과정 관찰 |
| ③ 파이프라인 표시 | 활용 및 적용 「좌표가 화면 효과로 바뀌는 단계」(카메라 입력→얼굴 찾기→특징점 좌표 계산→AR 효과 출력) | 4단계 흐름을 실시간으로 확인 |
| ③ 좌표 관찰 미션 | 직접 적용해 보기(왼쪽=X, 아래·위=Y) | 움직임 방향과 바뀌는 좌표의 관계 발견 |
| ① 안전 약속 / ④ 퀴즈 | 한 걸음 더!(얼굴은 개인정보), 토론(허락 없이 올리기), AI 탐험가 확인 문제 | 얼굴 정보 안전 태도, 배운 내용 확인 |

## 3. 화면·상호작용 상세

### ① 준비 — 안전 약속 (15강 `.promise` 패턴 재사용)
- 체크 3개 (모두 체크해야 「체험 시작하기」 활성화):
  1. "내 얼굴 화면은 **이 기기 화면에만 보이고 어디에도 저장·전송되지 않아요.**"
  2. "친구 얼굴을 찍거나, 찍은 화면을 **허락 없이 공유하지 않아요.** 얼굴은 소중한 개인정보예요."
  3. "체험이 끝나면 **카메라 끄기 버튼**을 눌러 카메라를 꺼요."
- 하단 안내: "카메라가 없어도 괜찮아요. 마우스로 마루 얼굴을 움직이는 연습 모드로 체험할 수 있어요."
- 시작 시 카메라 권한은 요청하지 않음(③ 진입 시점에 요청 — 권한 팝업의 맥락을 이해시키기 위해).

### ② 탐지 vs 식별 비교 체험 ★핵심 코너 (저자 특별 요청)
**"같은 장면, 다른 눈" — 모드 토글로 두 AI의 차이를 직접 조작하며 발견**

- 화면 구성 (위→아래):
  1. **모드 토글** (세그먼트 버튼 2개, 44px 이상): `[🔍 탐지 모드]` ↔ `[🪪 식별 모드]`. 선택된 쪽 purple 배경.
  2. **교실 장면 캔버스** (`<canvas>` 또는 absolute div, 4:3): 마루·소진 얼굴(등록됨), 처음 보는 학생 얼굴(미등록), 그리고 **화분**(얼굴 아님) 4개 오브젝트. 얼굴은 이모지/SVG 일러스트.
  3. **드래그 조작**: 마루 얼굴을 손가락/마우스로 잡고 움직일 수 있음("마루를 움직여 봐요!" 손가락 아이콘 힌트).
  4. **등록 명부 카드** (식별 모드에서만 표시): "이 문 앞 AI에 등록된 얼굴: 마루 🐻, 소진 🐰" — 식별은 '등록된 것과 비교'가 필요함을 시각화.
- **탐지 모드일 때**:
  - 얼굴 3개 모두에 **초록 네모 박스** + 박스 위에 좌표 라벨 `X:182 Y:96` (드래그하면 실시간 갱신).
  - 화분에는 박스 없음 → 말풍선 "얼굴이 아니라서 못 찾았어요".
  - 이름표 없음. 안내문: "탐지 모드는 얼굴이 **어디에** 있는지만 찾아요. 누구인지는 몰라요."
- **식별 모드일 때**:
  - 같은 장면에서 박스 색이 이름표로 바뀜: 마루 → `🪪 마루예요! (등록됨)`, 소진 → `🪪 소진이에요! (등록됨)`, 처음 보는 학생 → `❓ 등록 안 된 얼굴이에요`.
  - 좌표 라벨은 흐리게(회색) 유지 — "식별도 위치는 먼저 찾아요"를 암시.
  - 안내문: "식별 모드는 등록 명부와 비교해서 **누구인지**까지 알아내요."
- **비교표 자동 완성**: 두 모드를 각각 1번 이상 켜 보면 하단 비교표(구분/무엇을 하나요?/결과/사용처 — 교재 표 그대로)의 칸이 하나씩 채워지며 나타남. 둘 다 완료 시 "✅ 두 모드를 모두 관찰했어요!" + 「다음」 버튼 강조.
- 관찰 노트(`.note`): "🔍 오늘 ③단계에서 만들 고양이 귀 AR은 둘 중 **탐지**만 사용해요. 누구인지 몰라도 위치만 알면 스티커를 붙일 수 있어요!"

### ③ 고양이 귀 AR 미션 (카메라 + MediaPipe)
- 화면 구성:
  1. 「📷 카메라 켜기」 버튼 → 권한 요청 → 비디오 프리뷰(거울 반전) + 오버레이 캔버스.
  2. **좌표 패널** (비디오 아래, 큰 숫자 2개): `코 X: 421` `코 Y: 236` — 실시간 갱신, 값이 바뀔 때 살짝 하이라이트.
  3. **표시 옵션 토글 3개**: `[· 특징점 보기]` `[▢ 얼굴 박스]` `[🐱 고양이 귀]` — 특징점만 켜면 점 구름, 박스만 켜면 탐지 네모, 고양이 귀를 켜면 이마 위에 귀 스티커. 겹쳐 켜기 가능 → "점(특징점) → 숫자(좌표) → 스티커(AR)" 층위를 스스로 분해·조립.
  4. **스티커 선택**: 🐱 고양이 귀 / 🕶 선글라스(눈 특징점 기준) / 🎩 모자(교재 엔트리 실습과 동일: 코 위 보정값) — 3종.
  5. **미션 카드 3개** (달성 시 체크 애니메이션):
     - 미션1 "고개를 **왼쪽**으로 움직여 봐요 → 어느 좌표가 바뀌나요?" (X 변화 감지 시 "X좌표가 바뀌었어요!" 체크)
     - 미션2 "고개를 **아래로** 숙여 봐요 → Y좌표가 커지는 걸 확인해요" (Y 증가 감지 시 체크; "화면 좌표는 아래로 갈수록 Y가 커져요" 설명)
     - 미션3 "얼굴을 카메라에 **가까이** 대 봐요 → 귀가 커져요!" (얼굴 너비 증가 감지 시 체크)
  6. 「카메라 끄기」 버튼(coral) 상시 노출 — 누르면 스트림 stop + 트랙 해제 + "카메라를 껐어요. 얼굴 영상은 저장되지 않았어요." 메시지.
- 미션 3개 완료(또는 폴백 모드에서 대응 미션 완료) 시 「다음: 확인 문제 →」 강조.
- 탐험 실험 노트(`.note`): "🧪 얼굴을 손으로 반쯤 가려 봐요. 특징점이 흔들리거나 사라지죠? AI가 언제 얼굴을 못 찾는지 알 수 있어요."

### ④ 퀴즈 + 배지 (15강 `.quiz-opt` / `.badgebox` 패턴)
- 2문항 정답 시 「배지 받기 🏅」 활성화 → 배지 카드. (문항은 7절)
- 배지 카드 요약문: "오늘 배운 것: 카메라 입력 → 얼굴 찾기(탐지) → 특징점을 X·Y 좌표로 → 좌표 따라 스티커! 얼굴은 소중한 개인정보라 저장·공유 전에 꼭 허락을 받아요."

## 4. 파이프라인 / 시각화 설계

③단계 비디오 아래에 15강 `.pnode` 4노드 파이프라인을 배치, 매 프레임 상태 반영:

```
[1] 카메라 입력      → "화면이 들어와요" (스트림 활성 시 on)
[2] 얼굴 찾기(탐지)  → "얼굴 발견! ▢" / "얼굴을 찾는 중…" (faceLandmarks 존재 여부에 따라 ok/fail)
[3] 특징점 좌표 계산 → "코 X:421 Y:236" (실시간 숫자, 코 좌표와 동기화)
[4] AR 효과 출력     → "이마 좌표 + 보정값 위치에 🐱 그리기" (스티커 켜짐 시 ok)
```
- 얼굴을 가리면 [2]가 fail로 바뀌고 [3][4]가 꺼짐 → "찾기가 멈추면 뒤 단계도 멈춰요"를 체감.
- 좌표 패널 숫자는 6프레임마다 갱신(눈으로 따라갈 수 있는 속도), 스티커는 매 프레임(부드러움) — 관찰용과 렌더용 분리.
- 폴백 모드에서도 동일 파이프라인 사용, [1]만 "마우스 입력(연습 모드)"로 라벨 교체.

## 5. 폴백·예외 처리

| 상황 | 처리 |
|---|---|
| `getUserMedia` 미지원 / 권한 거부 / 카메라 없음 | **연습 모드 자동 전환**: 캔버스에 마루 얼굴 일러스트, 드래그(터치/마우스)로 이동. 얼굴 중심을 "코 좌표"로 간주해 좌표 패널·파이프라인·고양이 귀가 동일하게 반응. 미션 1·2·3도 드래그 좌·하 이동, 핀치/버튼 `[크게][작게]`로 대응 |
| MediaPipe CDN/모델 로드 실패 (오프라인 등) | 카메라 프리뷰는 두되 안내: "AI 모델을 못 불러왔어요. 연습 모드로 바꿔요" → 연습 모드 전환 버튼 |
| 모델 로딩 중 (2~5초) | 버튼에 스피너 + "AI가 얼굴 찾는 법을 준비하고 있어요…" (멈춘 것으로 오해 방지) |
| 얼굴 미검출 프레임 | 스티커·좌표 유지하지 않고 파이프라인 [2] fail 표시 + "카메라에 얼굴이 보이게 해 봐요" |
| 얼굴 2개 이상 | `numFaces:1`로 첫 얼굴만 사용, 안내: "한 명씩 체험해요" |
| 저사양 기기 프레임 드랍 | `requestAnimationFrame` 기반 + 검출은 비디오 타임스탬프 변경 시에만 수행 |
| 페이지 이탈/단계 이동 | ③을 떠나면 `stopCamera()` 자동 호출 (스트림 잔류 방지) |

## 8. 기술 구현 노트

- **CDN**: `https://cdn.jsdelivr.net/npm/@mediapipe/tasks-vision@0.10.14` (ESM `import { FaceLandmarker, FilesetResolver }`). 버전 고정(`@latest` 금지 — 재현성).
- **모델(.task) 파일**: 기본 경로가 `storage.googleapis.com/mediapipe-models/...face_landmarker.task`(외부 호스트)이므로, **레포 `/assets/face_landmarker.task`(약 3.7MB)로 self-host**하여 `modelAssetPath: '../assets/face_landmarker.task'` 사용 권장(허용 CDN 규칙 준수 + 구글 호스트 차단 학교망 대비). wasm은 jsdelivr에서 로드.
- **핵심 특징점 인덱스** (MediaPipe FaceMesh 478점): 코끝 `1`, 이마 중앙 `10`, 왼쪽 관자놀이 `103`, 오른쪽 관자놀이 `332`, 왼눈 `159`, 오른눈 `386`, 얼굴 폭 기준 `234`↔`454`.
- **핵심 로직 의사코드**:

```js
// 초기화
const vision = await FilesetResolver.forVisionTasks(JSDELIVR_WASM);
landmarker = await FaceLandmarker.createFromOptions(vision, {
  baseOptions:{ modelAssetPath: MODEL_PATH, delegate:'GPU' },
  runningMode:'VIDEO', numFaces:1 });

// 매 프레임
function loop(){
  if(video.currentTime !== lastT){
    lastT = video.currentTime;
    const res = landmarker.detectForVideo(video, performance.now());
    if(res.faceLandmarks.length){
      const lm = res.faceLandmarks[0];
      const nose = px(lm[1]), brow = px(lm[10]);
      const faceW = dist(px(lm[234]), px(lm[454]));      // 스티커 크기 기준
```
