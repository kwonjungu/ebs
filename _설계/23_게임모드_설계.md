# 23강 객체 탐지 게임 모드 — "물건 헌트" 설계·기획

## 1. 콘셉트
이모지 헌트 스타일. 라운드마다 **목표 물건**이 뜨고, 카메라로 그 물건을 비춰 **신뢰도(유사도) 60% 이상**으로 탐지하면 라운드 클리어. 테마는 **🏫 학교 / 🏠 집** 중 선택.

- 기존 23강 체험(자유 탐지)은 그대로 두고, 상단에 **[체험 모드 | 게임 모드]** 토글을 추가.
- 카메라·MediaPipe(Lite2) 파이프라인 재사용. 게임 모드에선 "모든 박스"가 아니라 **목표 물건만** 강조.

## 2. 화면 흐름 (상태)
1. **인트로**: 게임 제목 → 테마 선택(학교/집 큰 버튼) → [게임 시작]
2. **플레이**: 카메라 뷰 + 상단 HUD
   - 라운드 표시 `1/5`, 목표 칩 `📖 "책"을 찾아라!`
   - **찾음 게이지**: 목표 물건의 현재 최고 신뢰도(0→60 목표선). 60 넘으면 채워지고 클리어.
   - (선택) 라운드 타이머
3. **라운드 클리어**: ✓ 펑! + 점수 +, 1.2초 후 다음 라운드 자동
4. **결과**: 클리어 수 / 걸린 시간 / 별점(⭐×3) → [다시 하기] [테마 바꾸기]

## 3. 핵심 규칙
- 한 게임 = **5라운드**. 라운드마다 테마 풀에서 목표를 랜덤(중복 없이) 뽑음.
- 클리어 조건: 목표 라벨이 **신뢰도 ≥ 0.60** 으로 탐지 + **0.4초 유지**(깜빡임 방지).
- 게이지는 실시간 최고 신뢰도를 보여줘 "조금만 더!" 피드백.
- 목표가 아닌 물건이 잡혀도 게이지엔 반영 안 함(목표만).
- 사람(person)은 게임에서도 박스 없이 "얼굴은 비추지 않기" 유지.

## 4. 테마별 목표 풀 (모델이 잘 잡고 교실/집에서 찾기 쉬운 것)
**🏫 학교**: 📖 책 · ✏️ 가위 · ☕ 컵 · ⏰ 시계 · 🎒 가방 · 🪑 의자 · 🍶 병 · 📱 휴대전화 · ⌨️ 키보드 · 🖱️ 마우스
**🏠 집**: ☕ 컵 · 🍶 병 · 🥣 그릇 · 📺 리모컨 · ⏰ 시계 · 🧸 곰인형 · 🪴 화분 · 📖 책 · 🍎 사과 · 🍌 바나나

(각 풀에서 5개를 뽑음. 작아서 잘 안 잡히는 가위·숟가락 등은 제외/최소화.)

## 5. 점수·별점
- 라운드 클리어 = 20점. 남은 시간 보너스(타이머 모드일 때).
- 별점: 5개 다 클리어 ⭐⭐⭐ / 3~4개 ⭐⭐ / 1~2개 ⭐.
- 결과 화면에 "잘 찾은 물건" 목록 + 못 찾은 물건은 "다음엔 이렇게 비춰봐요" 팁.

## 6. 교육적 연결
- 라운드 실패/고전 시: "멀리 있거나 가려지면 AI가 잘 못 찾아요" → 23강 한계 학습과 연결.
- 결과 후 기존 **"한계 한 마디 적어보기"** 로 자연 연결.

## 7. 구현 메모
- 새 `<section id="game">`: 인트로/플레이/결과 3개 하위 화면 show/hide.
- 상태: `mode('play'|'game')`, `theme`, `roundQueue[]`, `roundIdx`, `targetKey`, `bestScore`, `holdTimer`, `score`, `startTime`.
- 탐지 루프에서 게임 모드일 때: detections에서 목표 라벨 최고 score 추출 → 게이지 갱신 → ≥0.6 유지 시 clearRound().
- 카메라/detector 로드·정리 로직 재사용(중복 최소화). reduced-motion·44px·aria 유지.
- 목표 물건에만 초록 박스+라벨, 나머지는 흐리게(선택).

## 8. 확정 필요 (기획 결정)
- **타이머**: (A) 시간 제한 없음, 정확도·개수로 별점 / (B) 라운드당 제한시간(예 20초) 스릴형 / (C) 전체 1개 타이머
- 라운드 수(기본 5), 신뢰도 기준(기본 60) 유지 여부
