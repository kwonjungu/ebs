- 포인트 컬러: **mint** (`--mint: #34D399`, `--mint-light: #D1FAE5`)
- 페이지 제목: `내 얼굴이 열쇠! 얼굴 인식 문지기 — 중급 17강`
- 히어로: 칩 "AI 탐험대 · 중급 17강" / h1 "🚪 내 얼굴이 열쇠! 얼굴 인식 문지기" / 서브 "가상 얼굴 카드로 유사도와 기준값을 실험해요. 카메라는 쓰지 않아요."
- **카메라·마이크 전혀 사용 안 함** — 교재 '도전! 가상의 얼굴 카드로 실험하기'를 그대로 웹으로 옮긴 안전 설계.

## 1. 학습 목표 연결 (교재 코너 대응)

| 체험 웹 단계 | 교재 코너 | 학습 목표 |
|---|---|---|
| ② 특징값 비교 | 개념 쏙쏙(특징점→특징값→유사도), 용어 쏙쏙, 활용 및 적용 「등록–비교–판단」 표 | 얼굴을 통째가 아닌 특징값(눈 사이 거리·코 위치·얼굴 너비) 숫자로 비교함을 이해 |
| ② 카드 4장 | 직접 적용해 보기 표(마루 정면 94 / 모자 마루 76 / 어두운 마루 68 / 지훈 32) | 등록값과의 차이가 작을수록 유사도가 높음을 발견 |
| ③ 기준값 실험 | 도전!(기준값 75→85→90→65 실험표), 문항 4(느슨한 기준값의 위험), 생각하기('확인 필요'의 이유) | 기준값을 올리면 본인 거부, 내리면 타인 통과 문제를 스스로 발견 |
| ① 안전 약속 / ④ 퀴즈 | 토론(동아리실 얼굴 인식), 안전한 얼굴 인식 활용 가이드, AI 탐험가 확인 문제 | 얼굴은 바꾸기 어려운 민감한 개인정보임을 확인 |

## 2. 단계 구성 (진행 바)

```
① 준비 → ② 특징값 비교 → ③ 기준값 실험 → ④ 확인하고 배지 받기
```

## 3. 화면·상호작용 상세

### ① 준비 — 안전 약속
- 안내 배너(mint-light): "🔒 이 체험은 **진짜 얼굴을 쓰지 않아요.** 마루와 지훈이의 **가상 얼굴 카드**로 원리만 실험해요. 카메라를 켜지 않아요."
- 체크 3개:
  1. "얼굴은 비밀번호와 달리 **쉽게 바꿀 수 없는 개인정보**라는 걸 기억해요."
  2. "진짜 사람 얼굴 사진을 이런 실험에 **함부로 쓰지 않아요.**"
  3. "얼굴 인식은 **틀릴 수 있어요.** 결과가 이상하면 사람이 한 번 더 확인해요."
- 도입 스토리 카드: 동아리실 문 일러스트 + "등록된 얼굴을 확인하면 문이 열립니다" + 모자 쓴 마루가 거부당한 장면(생각 열기 요약) → "왜 같은 마루인데 안 열렸을까요? 직접 실험해서 알아내요!"

### ② 특징값 비교 체험 — "AI 문지기는 숫자로 봐요"
- 화면 구성:
  1. **등록 카드** (상단 고정, mint 테두리): 🐻 마루 등록 얼굴 — `눈 사이 거리 42 · 코 위치 58 · 얼굴 너비 80` "동아리실에 미리 저장된 값이에요."
  2. **얼굴 카드 4장** (2×2 그리드, 탭 가능한 카드 버튼, 44px 이상):
     - 😊 마루 정면 `43 · 57 · 81`
     - 🧢 모자 쓴 마루 `46 · 62 · 84`
     - 🌙 어두운 마루 `38 · 65 · 85`
     - 🙂 지훈 `55 · 48 · 91`
  3. 카드를 누르면 **비교 애니메이션** 순차 재생:
     - (a) 등록값 vs 카드값이 3줄 막대 비교로 나란히 표시, 각 줄에 `차이 1` `차이 4`… 배지가 톡 나타남 (차이 작으면 mint, 크면 coral).
     - (b) 차이 합계 카운트업 → **유사도 점수 게이지**(0~100 반원 게이지)가 차오름: 마루 정면 94, 모자 76, 어두움 68, 지훈 32.
     - (c) 점수 밑 한 줄 해설: "차이가 작을수록 점수가 높아요!" / 지훈 카드일 땐 "다른 사람이라 차이가 커요."
  4. 4장을 모두 눌러 보면 카드마다 점수 스티커가 붙고 「다음: 기준값 실험 →」 강조.
- 관찰 노트(`.note`): "🔍 모자 쓴 마루(76)와 어두운 마루(68)를 봐요. **같은 마루인데** 모자·조명 때문에 점수가 내려갔어요. 얼굴 인식이 틀릴 수 있는 까닭이에요."
- 한 발자국 더(접기/펼치기): "점수는 어떻게 계산했을까요? → `유사도 = 100 − 2 × (차이의 합)` 직접 검산해 봐요!" (교재 '한 발자국 더' 추론 활동 대응)

### ③ 기준값 실험 — "문지기의 엄격함을 조절해요"
- 화면 구성 (위→아래):
  1. **동아리실 문 스테이지** (15강 `.stage` 다크 패턴 응용): 문 일러스트 + 판정 램프. 통과=문 열림 애니메이션+mint 램프 "🟢 통과 — 어서 와요!", 확인 필요=amber 램프 "🟡 확인 필요 — 선생님이 한 번 더 확인해요", 거부=coral 램프+문 닫힘 "🔴 거부".
  2. **기준값 슬라이더** (`input[type=range]` min 65 / max 95 / step 1, 기본 75, 큰 썸): 현재 값 크게 표시 + 상태 라벨 — 65~72 "😌 느슨해요" / 73~82 "🙂 보통" / 83~89 "😐 엄격해요" / 90~95 "🧐 매우 엄격해요".
  3. **얼굴 카드 4장** (②와 동일 카드, 점수 표시된 상태): 카드를 누르면 그 카드가 문 앞으로 걸어가는 애니메이션 → 파이프라인 3노드 진행 → 판정.
  4. **실험 기록표** (교재 도전 5단계 표 재현): 행=기준값 75/85/90/65(프리셋 버튼 4개로 빠른 이동), 열=카드 4장. 해당 기준값에서 카드를 판정할 때마다 칸이 자동으로 채워짐(통과🟢/확인🟡/거부🔴). 슬라이더가 프리셋이 아닌 값이면 "자유 실험" 행에 기록.
  5. **발견 카드 2개** (조건 충족 시 팝 등장 — 체험으로 문제를 '발견'하게):
     - 기준값 ≥ 85에서 모자/어두운 마루가 거부되면 → "💡 발견! 기준값을 **높이면** 진짜 마루도 거부당해요. (본인 거부 문제)"
     - 기준값 ≤ 68에서 어두운 마루 등이 통과되면 → "💡 발견! 기준값을 **낮추면** 닮은 다른 사람도 통과할 수 있어요. (타인 통과 위험)" (지훈 32는 65에서도 거부 — "그래도 아주 다른 얼굴은 막아요" 부연)
  6. 발견 카드 2개를 모두 얻으면 미션 완료 → "그래서 딱 좋은 기준값을 정하는 게 어려워요. 애매하면 **사람이 확인**하는 단계가 필요해요!" + 「다음: 확인 문제 →」.
- 생각하기 노트(`.note`): "🧪 '확인 필요'는 왜 있을까요? 자동으로만 판단하면 진짜 마루가 억울하게 거부되거나 닮은 사람이 잘못 통과할 수 있어서, **애매할 땐 사람이 한 번 더** 봐요." (교재 '생각하기' 대응)

### ④ 퀴즈 + 배지
- 2문항 정답 시 배지. 배지 카드 요약문: "오늘 배운 것: 등록 → 특징값 비교 → 유사도 → 기준값 판정. 기준값은 높여도 낮춰도 문제가 생겨요. 얼굴은 바꿀 수 없는 개인정보니 저장 여부를 꼭 확인해요."

## 4. 파이프라인 / 시각화 설계

③단계 판정 시 15강 `.pnode` 3노드가 순차 점등(350ms 간격):

```
[1] 등록  → "저장된 마루 특징값 42·58·80" (항상 on)
[2] 비교  → "카드 특징값과 차이 계산 → 유사도 76점" (점수 카운트업, ok)
[3] 판단  → "76 ≥ 기준값 75 → 통과!" (판정색: ok/amber/fail)
```
- [3] 노드에 **부등식이 그대로 표시** (`유사도 ? 기준값`) — 판정이 마법이 아니라 숫자 비교임을 보여주는 핵심 시각화.
- 보조 시각화: **수직선(넘버라인) 바** — 0~100 가로 바에 카드 4장의 점수 점 4개 고정, 기준값 슬라이더와 연동된 세로선이 좌우로 움직임. 세로선 오른쪽=통과 영역(mint), 왼쪽 10칸=확인 필요(amber), 그 왼쪽=거부(coral). 슬라이더를 움직이면 어떤 카드가 영역을 넘나드는지 한눈에 보임 — 이 강의 대표 시각화.

## 5. 폴백·예외 처리

- 카메라·마이크·외부 CDN 미사용 → 하드웨어/네트워크 폴백 자체가 불필요(전 기기 동일 체험). 이것이 이 강의 설계 포인트.
- 예외 처리:
  - 슬라이더 터치가 어려운 저학년 대비 `[−]` `[+]` 스텝 버튼 병행 제공.
  - ③에서 카드를 누르지 않고 슬라이더만 움직이면: 마지막 판정 카드를 자동 재판정(즉각 피드백 유지). 판정 이력이 없으면 힌트 말풍선 "얼굴 카드를 눌러 문 앞에 세워 봐요".
  - 애니메이션 중 연타 방지: 판정 진행 중 카드 비활성(300~900ms).
  - `prefers-reduced-motion` 시 애니메이션 즉시 완료 처리.
  - 실험 기록표가 좁은 화면에서 넘칠 때 가로 스크롤(`overflow-x:auto`).

## 7. 퀴즈 2문항 + 배지

- **Q1.** 얼굴 인식 문지기가 [통과]를 정하는 방법은? *(교재 문항 1·2 변형)*
  - ① 카메라에 보이는 얼굴을 사진 한 장으로 통째로 외워서
  - **② 특징값(눈 사이 거리 등)을 등록값과 비교해서, 유사도가 기준값보다 높으면 통과** ✔
  - ③ 카메라 앞에 서는 사람은 누구나 통과시켜서
- **Q2.** 실험에서 기준값을 65로 **낮췄더니** 생긴 일은? *(교재 문항 4 변형 — 체험 결과 회상형)*
  - ① 마루 정면도 매번 거부됐다
  - **② 어두운 마루처럼 점수가 낮은 얼굴까지 통과해서, 닮은 다른 사람도 통과할 위험이 커졌다** ✔
  - ③ 유사도 점수가 사라졌다
- **배지: 「얼굴 인식 문지기」 🗝️** (배지 원 안 이모지 🗝️)

## 8. 기술 구현 노트

- **의존성 없음**: 외부 CDN 0개, 바닐라 JS + CSS만. 얼굴 카드는 이모지+CSS 일러스트(이미지 파일 없이도 동작, `/assets` 마루 이미지가 있으면 교체).
- **데이터와 핵심 로직 의사코드** (교재 수치와 완전 일치 검증 완료):

```js
const REG = { eye:42, nose:58, width:80 };                 // 등록된 마루
const CARDS = [
  { id:'front', name:'마루 정면',   emo:'😊', eye:43, nose:57, width:81 }, // → 94
  { id:'cap',   name:'모자 쓴 마루', emo:'🧢', eye:46, nose:62, width:84 }, // → 76
  { id:'dark',  name:'어두운 마루', emo:'🌙', eye:38, nose:65, width:85 }, // → 68
  { id:'jihun', name:'지훈',       emo:'🙂', eye:55, nose:48, width:91 }, // → 32
];
function similarity(c){                                     // 교재 4장 점수 94/76/68/32 재현
  const diff = Math.abs(c.eye-REG.eye) + Math.abs(c.nose-REG.nose) + Math.abs(c.width-REG.width);
  return Math.max(0, 100 - 2*diff);
}
function judge(score, threshold){                           // 교재 실험표(75/85/90/65) 전부 일치
  if(score >= threshold) return 'pass';                     // 통과
  if(score >= threshold - 10) return 'check';               // 확인 필요
  return 'reject';                                          // 거부
}
// 슬라이더: input 이벤트 → 넘버라인 세로선 이동 + (마지막 카드 재판정)
// 판정 시퀀스: setTimeout 체인으로 pnode 1→2→3 점등, 점수 카운트업(rAF 20프레임)
// 기록표: results[threshold][cardId] = judge(...) 저장 후 셀 렌더; 프리셋(75/85/90/65) 행 우선
// 발견 카드: judge 결과 로그를 검사 —
//   (t>=85 && card∈{cap,dark} && reject) → discover('strict');  본인 거부
//   (t<=68 && card==='dark' && pass)     → discover('loose');   타인 통과 위험
//   둘 다 발견 시 missionDone()
// 문 애니메이션: CSS transform(rotateY) + 판정색 램프 클래스 교체
```
- 단계 이동·퀴즈·배지·`.note`·`.pnode`는 15강 코드 패턴 재사용, 컬러만 mint로.
