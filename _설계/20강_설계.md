# 20강 체험 웹 설계서 — 내 상상 속 마스코트, AI와 함께 그려 볼까? (이미지 생성·프롬프트)

- 경로: `/20/index.html` (단일 파일, 바닐라 JS + 인라인 SVG, 서버·API 키 없음)
- 포인트 컬러: **amber** (`--amber: #FBBF24`, `--amber-light: #FEF3C7`)
- 콘셉트: **프롬프트 빌더 + SVG 마스코트 조합기**. 진짜 이미지 생성 AI 없이, "쓴 글(선택)이 곧 그림이 된다"를 파츠 조합으로 재현한다.
- 핵심 메시지: "AI는 내 머릿속이 아니라 **내가 쓴 글**을 읽고 그려요. 비워 두면 AI가 마음대로 채워요. 첫 그림은 시안 — 확인하고 수정 요청해요."

## 1. 학습 목표 연결

| 교재 학습 목표 | 체험 웹 대응 |
|---|---|
| ① 이미지 생성 AI가 그림을 만드는 흐름 이해 | 파이프라인 4노드(아이디어→프롬프트→AI가 읽고 그리기→확인·수정)를 매 생성마다 점등 |
| ② 막연한 요청 vs 구체적 요청의 차이 | ["멋진 마스코트 그려 줘"] 버튼 = 전 항목 랜덤(엉뚱) ↔ 빌더로 조립 = 의도대로. 나란히 비교 |
| ③ 결과 확인 후 수정 요청 | 시안 체크리스트 → 한 항목만 바꿔 [다시 그리기] = 수정 요청 루프 |
| ④ 안전·책임(개인정보·저작권) | 안전 코너 O/X 5문제(교재 표 그대로) + 금지 문구 상시 노출 |

## 2. 단계 구성 (진행 바 단계명)

```
① 준비  →  ② 막연 vs 구체  →  ③ 마스코트 공방  →  ④ 확인하고 배지 받기
```
- ①: 안전 약속 3개 체크(개인정보·저작권·시안 태도).
- ②: 개념 체험. 막연 버튼의 랜덤 참사 → 빌더로 한 칸씩 채우면 결과가 잡혀 가는 대비 체험.
- ③: 미션. 목표 카드(마루의 주문서)대로 마스코트 완성 → 체크리스트 → 수정 요청 루프 → 첫 시안 vs 최종 나란히 비교. + 안전 O/X 코너.
- ④: 퀴즈 2문항 + 배지.

## 3. 화면·상호작용 상세

### ① 준비
- 헤더: 칩 "AI 탐험대 · 중급 20강", 제목 "🎨 내 상상 속 마스코트, AI와 그려요", 부제 "프롬프트를 구체적으로 쓸수록 그림이 내 생각과 가까워져요."
- 도입 카드: "마루가 '멋진 마스코트 그려 줘!'라고만 했더니 무서운 거대 로봇이 나왔어요. 왜 그럴까요?"
- 안전 약속 3개 체크 → [체험 시작하기].

### ② 막연 vs 구체 (개념 체험)
화면 구성:
1. **캔버스 카드**: 260×260 SVG 미리보기(흰 캔버스 + 이젤 프레임 느낌). 처음엔 "❓ 아직 아무것도 없어요".
2. **막연 요청 버튼**(coral 계열): 「"멋진 마스코트 그려 줘" (막연한 요청)」— 누르면 전 카테고리 랜덤 조합 + 낮은 확률(30%)로 '엉뚱 파츠'(감자 몸통, 시무룩 표정) 등장. 생성된 프롬프트 표시줄엔 "멋진 마스코트 그려 줘" 그대로, 그 아래 amber 태그 5개: "대상? AI가 마음대로 / 행동? AI가 마음대로 / …".
3. **프롬프트 빌더**(카테고리 5줄, 칩 버튼 택1):
   - 대상: 🤖 로봇 / 🐱 고양이 / 🐦 새
   - 행동: 🏃 달리기 / 🦘 점프 / 👋 손 흔들기
   - 소품: 🎩 모자 / ⭐ 별 배지 / 🎒 탐험 가방
   - 스타일: ✏️ 만화풍 / 🟩 픽셀풍
   - 분위기: ☀️ 밝은 / 🌙 차분한
4. **완성 프롬프트 문장**(실시간 조립, `.sttbox` 스타일 재사용): 선택할 때마다 문장이 갱신되고 방금 바뀐 조각이 하이라이트. 예: "**운동화를 신고 달리는 로봇** 마스코트를, **별 배지**를 달고, **만화풍**으로, **밝은** 분위기로 그려 줘."
   - 미선택 칸은 물음표로: "( ? ) 마스코트를 …" — 미선택 상태로 [그리기]를 누르면 그 칸만 랜덤 + 캔버스에 amber 점선 테두리 태그 "이 부분은 AI가 마음대로 채웠어요!"
5. **[그리기] 버튼**(amber 대형 버튼) → 4절 파이프라인 연출 → 캔버스에 마스코트 등장.
6. 관찰 노트: "막연 버튼을 3번 눌러 봐요. 매번 다른 엉뚱한 그림이 나와요. 이번엔 다섯 칸을 모두 채우고 그려 봐요. 어느 쪽이 내 생각과 가까운가요?"

### ③ 마스코트 공방 (미션)
1. **주문서 카드**(교재 마루의 아이디어): "🎯 목표 — 별 배지를 단, 달리는 로봇을, 만화풍으로, 밝은 분위기로!" (소품=별 배지, 대상=로봇, 행동=달리기, 스타일=만화풍, 분위기=밝은)
2. 빌더로 조립 → [시안 만들기] → 캔버스에 시안 1 표시(작은 썸네일로 히스토리 보관).
3. **확인 체크리스트**(교재 체크리스트 축약, 자동 판정): "대상이 로봇인가요? / 별 배지가 보이나요? / 만화풍인가요? / 밝은 분위기인가요? / 달리고 있나요?" — 선택과 목표를 비교해 ✔/✘ 자동 표시.
4. ✘가 있으면 **수정 요청 패널** 등장: "좋은 점은 그대로 두고, 고칠 곳만 콕 집어요!" → ✘ 항목 옆 [이 부분 고치기] 버튼 → 해당 카테고리 칩만 깜빡이며 스크롤 → 바꾸고 [다시 그리기]. (= 교재의 작성→확인→수정 3단계 루프)
5. 전부 ✔이면 **비교 카드**: 시안 1(첫 그림)과 최종본을 나란히 표시 + "첫 그림은 완성품이 아니라 **시안**! 확인하고 고쳐서 점점 좋아졌어요."
6. **안전 코너 — O/X 5문제**(교재 표 그대로, 버튼 O/X):
   | 요청 내용 | 정답 |
   |---|---|
   | 내 상상 속 로봇 마스코트를 만화풍으로 그려 달라고 하기 | O |
   | 친구 얼굴 사진을 올려 마스코트로 바꿔 달라고 하기 | X |
   | 실제 우리 학교 이름·로고·주소를 그림에 넣어 달라고 하기 | X |
   | 유명 만화 캐릭터를 똑같이 따라 그려 달라고 하기 | X |
   | 완성한 그림이 목적에 맞는지, 이상한 부분은 없는지 확인하기 | O |
   - 5문제 완료해야 [다음: 확인 문제] 활성화. 오답 시 한 줄 이유 표시(예: "친구 얼굴은 개인정보예요. 함부로 올리지 않아요.").

### ④ 퀴즈 + 배지
7절 참고. 배지 화면에는 내가 만든 최종 마스코트 SVG를 배지 원 안에 넣어 준다(나만의 배지 감성).

## 4. 파이프라인/시각화 설계

15강 `.pipe/.pnode` 패턴 재사용, [그리기]마다 순서 점등(각 350ms):

```
1. 아이디어를 떠올려요     → (주문서/내 선택)
2. 프롬프트를 작성해요     → "달리는 로봇 마스코트를, 별 배지를 달고, 만화풍으로, 밝게"
3. AI가 글을 읽고 그려요   → 프롬프트 속 키워드가 하나씩 하이라이트되며 캔버스에 해당 파츠가 순서대로 그려짐
                             (로봇 몸통 → 달리기 포즈 → 별 배지 → 만화풍 마감 → 밝은 배경)
4. 확인하고 수정을 요청해요 → 체크리스트 카드로 연결
```
- **노드 3이 핵심 시각화**: 키워드 하이라이트와 파츠 등장을 1:1로 동기화 → "쓴 글이 곧 그림이 된다"가 눈에 보임. 미선택(랜덤 채움) 파츠는 등장할 때 amber 점선 + "🤖 마음대로!" 말풍선.
- 파츠 등장 애니메이션: opacity+scale 0.2s, 파츠당 300ms 간격(총 ~1.5s).
- 캔버스 하단 프롬프트 띠: 최종 문장이 항상 그림 아래 붙어 "이 글 → 이 그림" 대응 강조.

## 5. 폴백·예외 처리

- **외부 의존 0**: 이미지 파일 없이 전부 인라인 SVG → 로딩 실패 없음. 카메라·마이크·CDN 불필요.
- SVG 미지원 브라우저는 사실상 없으나, 방어적으로 `<svg>` 내부에 텍스트 폴백("그림을 표시할 수 없어요") 포함.
- 연출 중 [그리기] 연타 방지(진행 중 버튼 disabled). 카테고리 칩은 언제든 변경 가능하되 그리는 중에는 큐잉하지 않고 무시.
- 미선택 + [그리기]: 에러가 아니라 학습 장치(랜덤 채움 + 표식). 즉 "빈 값 예외"를 개념 체험으로 전환.
- `prefers-reduced-motion` 시 파츠 순차 등장 생략, 완성본 즉시 표시.
- 좁은 화면(320px~): 캔버스 위, 빌더 아래 세로 배치. 칩 버튼 min-height 44px, 가로 스크롤 없이 줄바꿈.
- 텍스트 입력창 없음 → 개인정보 입력 경로 원천 차단.

## 7. 퀴즈 + 배지

**Q1.** 생성형 AI 이미지 도구는 무엇을 보고 그림을 그릴까요?
- ① 내 머릿속 생각을 직접 읽어서 그려요 (✘)
- ② 내가 입력한 글(프롬프트)을 읽고 그려요 (✔)
- ③ 인터넷의 그림을 그대로 복사해서 보여 줘요 (✘)

**Q2.** AI가 만든 첫 번째 그림을 대하는 알맞은 태도는?
- ① 마음에 안 들면 곧바로 포기해요 (✘)
- ② 유명 캐릭터랑 똑같이 그려 달라고 다시 요청해요 (✘)
- ③ 시안이라고 생각하고, 고칠 점을 콕 집어 다시 요청해요 (✔)

**배지**: 🎨 **프롬프트 디자이너** — "오늘 배운 것: AI는 내가 쓴 글을 읽고 그려요. 대상·행동·소품·스타일·분위기를 구체적으로! 첫 그림은 시안, 개인정보·저작권은 꼭 지켜요." (배지 원 안에 내가 만든 마스코트 표시)

## 8. 기술 구현 노트

### 8-1. 데이터 구조 — 카테고리·SVG 파츠 목록

```js
// SVG는 viewBox 0 0 200 200 공통 좌표계. 파츠는 <g id> 조각 문자열(또는 빌더 함수)로 보관.
const CATS = {
  subject: { label: '대상', opts: [
    { id: 'robot', name: '로봇',   emoji: '🤖', phrase: '작은 탐험가 로봇' },
    { id: 'cat',   name: '고양이', emoji: '🐱', phrase: '아기 고양이' },
    { id: 'bird',  name: '새',     emoji: '🐦', phrase: '파랑새' },
  ]},
  action: { label: '행동', opts: [
    { id: 'run',  name: '달리기',    phrase: '운동화를 신고 달리는' },
    { id: 'jump', name: '점프',      phrase: '높이 점프하는' },
    { id: 'wave', name: '손 흔들기', phrase: '손을 흔드는' },
  ]},
  prop: { label: '소품', opts: [
    { id: 'hat',   name: '모자',      phrase: '모자를 쓰고' },
    { id: 'badge', name: '별 배지',   phrase: '별 배지를 달고' },
    { id: 'bag',   name: '탐험 가방', phrase: '탐험 가방을 메고' },
  ]},
  style: { label: '스타일', opts: [
    { id: 'toon',  name: '만화풍', phrase: '귀여운 만화풍으로' },
    { id: 'pixel', name: '픽셀풍', phrase: '픽셀 아트풍으로' },
  ]},
  mood: { label: '분위기', opts: [
    { id: 'bright', name: '밝은',   phrase: '밝고 친근한 느낌으로' },
    { id: 'calm',   name: '차분한', phrase: '차분하고 잔잔한 느낌으로' },
  ]},
};

// SVG 파츠 설계 (레이어 순서: 배경 → 몸통 → 포즈(팔다리) → 소품 → 표정)
// - 몸통 3종 × 스타일 2종 = 6조각: toon은 rx 큰 rounded rect/원+부드러운 스트로크,
//   pixel은 10px 격자 <rect> 모자이크(shape-rendering: crispEdges).
// - 행동은 파츠 교체가 아니라 "팔다리 조각 + transform"으로 처리:
//   run  = 몸통 기울기 rotate(-8) + 다리 벌림 + 뒤로 속도선 3개
//   jump = translateY(-14) + 다리 모음 + 아래 그림자 타원 + 반짝 별 2개
//   wave = 한 팔 위로(rotate) + 손끝 ✨
// - 소품: hat(머리 위 사다리꼴+챙), badge(가슴 위 5각 별, amber), bag(등 뒤 사각+어깨끈)
// - 분위기: bright = 배경 원형 그라데이션(#FEF3C7)+해✳, 웃는 눈(곡선)
//           calm   = 배경(#EBF3FF)+달·별, 잔잔한 눈(직선에 가까운 곡선)
// - 엉뚱 파츠(막연 요청 전용): potatoBody(갈색 감자 몸통), scaryFace(찡그린 눈썹),
//   giant(scale 1.35로 캔버스를 뚫을 듯) — 30% 확률로 1개 섞임.
const PARTS = {
  body:  { robot: { toon: SVG_STR, pixel: SVG_STR }, cat: {...}, bird: {...} },
  pose:  { run: {...}, jump: {...}, wave: {...} },        // 대상별 오프셋 테이블 참조
  prop:  { hat: SVG_STR, badge: SVG_STR, bag: SVG_STR },
  bg:    { bright: SVG_STR, calm: SVG_STR },
  weird: { potatoBody: SVG_STR, scaryFace: SVG_STR, giant: 'transform' },
};
```

### 8-2. 핵심 로직 의사코드

```
state.sel = { subject:null, action:null, prop:null, style:null, mood:null }

onChipClick(cat, id):
  state.sel[cat] = id
  renderPrompt()              // phrase들을 문장 틀에 끼움, 방금 조각 하이라이트
  // 문장 틀: `${action} ${subject} 마스코트를, ${prop}, ${style}, ${mood} 그려 줘.`
  // 미선택은 "( ? )" 표시

onDraw(vague = false):
  if (drawing) return; drawing = true
  filled = {...state.sel}; autoFilled = []
  for (cat of CATS):
    if (vague || !filled[cat]):
      filled[cat] = randomOpt(cat); autoFilled.push(cat)   // AI가 마음대로 채움
  weird = vague && Math.random() < 0.3 ? pickWeird() : null
  runPipe([
    node1(주문서/선택 요약),
    node2(프롬프트 문장 — vague면 "멋진 마스코트 그려 줘" 그대로),
    node3(파츠 순차 렌더: bg(mood) → body(subject×style) → pose(action)
          → prop → face(mood), autoFilled 파츠엔 점선+"마음대로!" 태그, weird 적용),
    node4(체크리스트 카드 표시),
  ])
  history.push(svgSnapshot()); drawing = false

renderMascot(filled, weird):
  svg.innerHTML = PARTS.bg[filled.mood]
    + (weird==='potatoBody' ? PARTS.weird.potatoBody
                            : PARTS.body[filled.subject][filled.style])
    + PARTS.pose[filled.action].for(filled.subject)
    + PARTS.prop[filled.prop]
    + face(filled.mood, weird==='scaryFace')
  if (filled.style==='pixel') svg.classList.add('pixel')   // crispEdges + 격자 필터

checklist(goal, filled):                      // ③미션 자동 판정
  return CATS.keys.map(cat => ({ cat, ok: filled[cat] === goal[cat] }))
```
